## Summary

席位制 SaaS 并非全面死亡,而是结构性失灵于知识工作型 SaaS。上市公司一分为二:抗代理席位(身份、可观测、协作、合规)继续按席位复利;可被代理替代席位(一线客服、SDR、初级开发、初级法务)正在崩塌为按结果/按用量定价(Salesforce Agentforce $2/对话、Intercom Fin $0.99/解决、Zendesk per-resolution、Klarna 替换 700 客服)。估值范式 EV/Revenue × Rule-of-40 仍在,但分母从'席位×ARPU'切换为'用量×take-rate',机械性带来 6–8pp 毛利压缩、并把混合增速降至约 20%。我估计可替代名单仍有 15–25% 倍数下修空间未被 price-in,抗代理复利者中性偏正,纯结果定价的 AI-native 垂直 agents 将形成新溢价品类。

## Report

# 卡片 01 — Agentic 时代的 SaaS 估值终局
**分析师:** TMT行业分析师 · **立场:** initial · **日期:** 2026-05-01

## 核心判断 (Thesis)
"按席位收费(per-seat)"并未**普遍死亡**,而是**在知识工作型 SaaS 中结构性失灵**,并把上市公司一分为二:一类是"抗代理席位"(协作、真实记录、身份、可观测性)——席位仍能复利;另一类是"可被代理替代席位"(一线客服、SDR/BDR、初级开发、初级法务、基础分析师)——席位正在崩塌为按结果/按用量定价。SaaS 估值范式 NTM EV/Revenue × Rule-of-40 并未崩盘,但**分母正在变形**:从"席位 × ARPU"切换为"用量 × take-rate",从而机械性地压缩毛利、并对整个板块重定价。

## 1. 经验证据:边际上,席位制已经在死

过去 18 个月,Agent 时代的定价试验菜单已经清晰,**没有一项是按席位**:

| 厂商 | 产品 | 计价单元 | 含义 |
|---|---|---|---|
| Salesforce | Agentforce / Agentforce 2.0 | 约 **$2 / 次对话** | 替代约 $165/席位的 Service Cloud 用户 |
| Intercom | Fin AI Agent | **$0.99 / 次解决** | 直接替代一线客服席位 |
| Zendesk | AI Agents | 按 resolution、分档 | 收入彻底脱钩于人头 |
| ServiceNow | Now Assist | **混合**:席位 + 用量 credits | 保住表观 ARR,同时按价值计量 |
| Microsoft | Copilot Studio | **按消息**($0.01–$0.20) | Studio 走用量;M365 Copilot 仍 $30/席位 |
| GitHub | Copilot | $19 / $39 / **agent SKU** | 席位档保留,agent SKU 是用量 |
| Klarna | OpenAI 合作(2024) | 替换约 **700 名客服**,年化节省约 $40M | 需求侧证据:客户会主动砍席位 |
| Workday | Agent System of Record | 按 agent 授权,**agent ≠ seat** | 为非人类重新发明了"席位"抽象 |

规律是:**厂商正在抢在续约前发明非席位的价值单位**,以免客户机械性削减席位。Agentforce、Fin、Zendesk 的 resolution 定价是先行指标;Microsoft 的双轨——助手按席位、平台按用量——正成为行业收敛形态。

## 2. 知识工作型 SaaS 为何会崩

按席位授权依赖三条隐含假设:(i) 价值随用户线性增长;(ii) 人是生产单位;(iii) 席位数随客户人头一起涨。Agent 把这三条都打破:

- **价值与人头脱钩。** Klarna 700 席位的崩塌说明:当 Agent 吸走整个工作流,客户支付意愿(WTP)会从"按剩余人头授权"迁移到"按结果付费"。席位收入会渐近于残留的"监督人头",一个体量小、增速慢的数字。
- **NRR(净留存率)——SaaS 质量核心指标——被重新装管。** 历史上 120%+ 的 NRR 来自席位扩张 + 价格抬升。Agent 时代的扩张迁移到**用量增长**,而用量更顺周期、更弹性、更与客户自身业务量相关——也就是携带了纯席位制从未有过的经营杠杆风险。
- **毛利率被压缩。** 纯席位 SaaS 通常 78–82% GM,因为算力占 COGS 比重小。Agentic 工作负载把推理成本转嫁过来——即使模型成本以约 70% YoY 速度下降,成熟态结构性下限仍是 55–65% GM。这是市场尚未充分消化的最大估值变量。

## 3. 分化:哪些席位活下来

并非所有席位都可被替代。一份可防御的分类:

**抗代理(席位继续存在,可能有 AI 加价):**
- 身份与访问(Okta、CyberArk)——每个人和每个 agent 都需要凭证,agent 反而**扩大**单元数。
- 平台层可观测/开发工具(Datadog、Snowflake)——本就大体按用量计价,agent 是净需求增量。
- 协作/真实记录(Atlassian、Notion、Figma、Slack)——席位**就是**社交图谱;agent 提升人的吞吐,但不替代人在工作流中的存在。
- 垂直合规/受监管工作流(Veeva、nCino)——人的签字本身就是产品。

**可被代理替代(席位崩塌,定价必须迁移):**
- 一线客服(Zendesk、Freshworks、Intercom 旧 SKU)
- SDR / BDR / 外呼销售(Salesforce 大额 SKU 暴露)
- 初级 IDE 席位(Copilot 已在双档化)
- 文档审阅/初级法务(DocuSign CLM、Ironclad)
- 基础数据录入/财务运营(Workday 低档、ServiceNow 中档)

可投资的真问题是**每家上市公司内部的结构占比变化**,而不是 SaaS 这个品类是否消亡。

## 4. 这对估值倍数意味着什么

为 Agent 时代 SaaS 厂商建立的简化框架:

- **收入结构:** 假设 60% 旧席位 ARR(增速 5–8%)+ 40% 新用量 ARR(增速 40–60%)。混合增速约 20%——比上一周期纯席位时代的多数大盘 SaaS **更低**。
- **毛利率:** 旧席位 80% × 0.6 + 用量 60% × 0.4 = **72%** 混合,对比 Agent 前的 78–80%,**结构性 GM 压缩 6–8 个百分点**。
- **Rule-of-40:** 若 FCF 利润率维持 25%,R40 = 45%——仍然健康。但 2024–25 年已发生的倍数收缩(NTM EV/Revenue 从 2021 年高点约 15× 到中位数 6–7×)只 price 进了大约一半,另一半即 GM 压缩仍在被吸收。
- **净效应:** 我估计可被代理替代的纯种公司还有 **15–25% 的倍数下修**(传统 CCaaS、中端 CRM、合同生命周期);抗代理复利者(身份、可观测、开发平台)倍数**持平至上修**——因为用量对它们是**叠加**而非**替代**。

## 5. 反方观点 / 我可能错在哪

- **Agent 自身 land-and-expand。** 若"Agent-as-Employee"定价(Workday Agent SoR、Salesforce "数字劳动力"框架)被接受,厂商可以通过卖 agent license 把丢掉的席位收回来。逻辑成立,但需要客户接受单个 agent 约 $2k/月的报价——早期企业数据看,他们不会。
- **推理成本崩塌跑赢价格侵蚀。** 若前沿推理再降一个数量级(DeepSeek-V4 节奏),GM 压缩反转,用量定价反成为利润顺风。
- **结果定价的反身性。** 客户一旦发现厂商拿走 30%+ 的价值,会反向压价回到固定/容量定价——换个名字部分恢复席位制经济逻辑。

## 6. 给整个研究线的结论

席位制 SaaS 在可被代理替代的子集中**已不再是增长的主导单位**,但它在整个 SaaS 全集中仍是**底盘收入的主导单位**。估值真问题因此不是"SaaS 还活不活",而是"**面对一个增长来自结构性更低毛利、更顺周期、更与客户业务量相关的收入流的生意,合理倍数应是多少**"。我的初步估计:可替代名单还有 -15–25% 的倍数压缩(超出已 price-in 部分);抗代理复利者中性偏正;并将出现一个**新的溢价品类**——从第一天就纯按结果定价的 AI-native 垂直 agents(Harvey、Sierra、Decagon)。
