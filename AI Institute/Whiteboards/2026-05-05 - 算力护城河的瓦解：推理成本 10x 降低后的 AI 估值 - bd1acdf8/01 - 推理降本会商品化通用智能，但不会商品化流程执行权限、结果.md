# 推理成本不再是护城河：AI 应用公司的 Alpha 将迁移到哪里

**角色：** 首席策略师  
**Board：** bd1acdf8-9e8b-419b-bd45-e937d86d8f80  
**Card：** 1/8  
**问题：** 如果推理成本不再是门槛，AI 应用公司的 Alpha 应来自算法效率、数据孤岛还是任务执行的闭环能力？

## 核心判断

我对初始问题的策略结论是：在推理成本进一步下降 10x 之后，AI 应用公司的可投资 Alpha 首先来自**任务执行闭环能力**，其次来自**专有运营数据**，最后才是单独的**算法效率**。

原因很直接。推理成本下降会把 API 层的通用智能逐步商品化，但不会同步商品化企业流程中的执行权限、执行之后产生的数据回流，以及允许软件真正完成任务所需的治理体系。放到 A 股策略语境里，主题重心应从“谁能更便宜调用模型”转向“谁掌握了可重复的业务流程，让 AI 能观察、决策、执行并学习”。

我对未来 12-24 个月 Alpha 来源的工作权重如下：

| Alpha 来源 | 权重 | 投资含义 |
|---|---:|---|
| 任务执行闭环能力 | 50% | 质量最高的 Alpha：掌握流程、执行权限、结果反馈和可衡量 ROI。 |
| 专有数据孤岛 | 30% | 只有当数据具备独占性、鲜活度、合法使用权，并与动作结果相连时才可持续。 |
| 算法效率 | 20% | 对利润率和延迟重要，但若不能嵌入专有产品闭环，容易被竞争抹平。 |

## 证据基础

成本冲击是真实存在的。Stanford AI Index 2025 显示，达到 GPT-3.5 级别 MMLU 表现的模型查询成本，从 **2022 年 11 月每百万 tokens 20.00 美元**降至 **2024 年 10 月每百万 tokens 0.07 美元**，约 18 个月下降**超过 280 倍**；按任务不同，LLM 推理价格每年下降 **9 到 900 倍**。[^1]

模型层价格也越来越透明。在本轮研究访问到的 OpenAI 公开 API 定价页中，GPT-5.5 价格为 **每 1M input tokens 5.00 美元**、**每 1M cached input tokens 0.50 美元**、**每 1M output tokens 30.00 美元**；GPT-5.4 价格为 **每 1M input tokens 2.50 美元**、**每 1M cached input tokens 0.25 美元**、**每 1M output tokens 15.00 美元**。[^2] 无论下一轮 10x 降本来自模型架构、缓存、小型专用模型还是竞争，方向已经明确：token 成本正在变成可管理的变动成本，而不是战略护城河。

DeepSeek 说明算法效率仍然重要，但它本身并不足以成为应用公司的护城河。DeepSeek-V3 技术报告描述了一个 **671B** 参数的 Mixture-of-Experts 模型，每个 token 激活 **37B** 参数，并用 MLA 与 DeepSeekMoE 提升推理和训练效率；报告还披露了 **14.8T** 预训练 tokens 与 **2.788M H800 GPU hours** 的完整训练消耗。[^3] 这是模型层工程成就。对应用公司而言，可投资的问题不是能否复刻 DeepSeek 的工程，而是能否把更便宜的推理转化为可复购、可收费的任务完成。

中国政策也更指向应用渗透，而不是单纯的算力稀缺。国务院“AI+”行动提出，到 **2027 年**新一代智能终端和智能体渗透率超过 **70%**，到 **2030 年**超过 **90%**，重点领域包括科技、产业、消费、民生、治理和全球合作。[^4] 同一政策也强调数据供给、模型能力、高效训练和推理、开源生态、人才以及场景化落地。对 A 股而言，这支持更广泛的应用周期，但也会因为政策支持吸引拥挤资金，从而提高差异化门槛。

企业采用数据强化了同一逻辑。McKinsey 2025 全球 AI 调研显示，**88%** 的组织表示至少在一个业务职能中经常使用 AI，但只有约三分之一开始在企业层面规模化 AI；**23%** 的组织正在企业某处规模化 agentic AI 系统，另有 **39%** 正在试验 AI agents。[^5] McKinsey 的 agentic AI 报告指出，横向 copilot 往往带来分散收益，而高影响力的垂直场景难以规模化，主要受流程、组织、数据和集成障碍制约；该报告把 agents 定义为结合记忆、规划、编排和集成来执行流程的系统。[^6]

## 三类护城河

### 1. 算法效率：必要，但多数时候只是毛利率杠杆

算法效率包括模型选择、微调、检索、路由、缓存、量化、蒸馏、上下文压缩，以及由评测驱动的 prompt 或工具优化。它能在 token 价格较高时保护毛利，也能通过降低延迟打开新的用户体验。在中国市场，由于国产芯片和算力约束仍然存在，模型与硬件协同优化仍具价值。

但当推理成本下降 10x 后，对多数 AI 应用公司而言，算法效率作为单独护城河的价值会下降。市场不会只因为某家公司能更便宜地完成同一个通用任务，就长期给予估值溢价。真正的溢价来自效率让公司做到竞争对手做不到的事：单次会话执行更多步骤、服务低延迟工作流、离线或端侧运行，或者以更低成本积累结果数据。

**A 股含义：** 不应为笼统的“AI 效率”叙事支付过高估值，除非它能连接到可见收入转化。更应偏好效率绑定高频流程、端侧部署、监管或延迟要求、专有评测体系的公司。

### 2. 数据孤岛：只有运营化、鲜活且有授权时才可持续

数据仍是护城河，但定义需要收紧。静态文档库、通用行业语料和抓取来的公开数据，会随着基础模型能力提升和检索工具标准化而失去稀缺性。更有价值的数据护城河是运营数据，并且需要满足：

| 检验标准 | 重要原因 |
|---|---|
| 独占性 | 竞争对手无法低成本购买或抓取同类数据。 |
| 鲜活度 | 数据频繁变化，实时访问本身有价值。 |
| 授权清晰 | 公司具备法律和客户层面的使用权限。 |
| 与动作相连 | 数据记录决策结果，而不仅是输入上下文。 |
| 足够结构化 | 能进入评测、自动化和产品改进循环。 |

例如，医院 IT 厂商的历史资料库如果只是文本，价值有限；如果它能连接诊断、治疗路径、医保审核、患者随访和结果，价值就显著提升。工业软件厂商的 CAD 或 MES 数据如果只用于文档搜索，价值有限；如果 AI 能调整排产、工艺参数、质检和维护动作，并从良率或停机时间中学习，价值就高得多。

**A 股含义：** 不能把所有“数据丰富”行业一概而论。估值溢价应给那些能证明独占数据访问和持续使用权的软件与平台公司，尤其是工业软件、医疗 IT、金融 IT、物流、能源管理以及政府和企业治理系统。

### 3. 任务执行闭环：最高质量的 Alpha

任务执行闭环是最强护城河，因为它把前两者结合起来。一个闭环意味着 AI 系统能够：

1. 观察任务上下文。
2. 决策或给出行动建议。
3. 通过企业系统、设备或人机协同流程执行动作。
4. 衡量结果。
5. 将结果反馈到下一轮决策。

这是 AI 应用公司真正能防守定价的地方。客户不会为更便宜的推理本身支付高倍数估值；客户愿意为服务周期缩短、转化率提升、缺陷率下降、回款加快、合规错误减少、停机时间降低或劳动生产率提升付费。掌握闭环的公司可以按业务价值定价，而不是按 token 消耗定价。

闭环也会带来复利优势。每一次完成任务都会产生带标签的结果数据，暴露异常场景，改善流程图谱，并提高切换成本。这与 chatbot wrapper 有本质差异，后者可能有使用量，但通常不掌握经济事件本身。

**A 股含义：** 行业配置应偏向已经位于关键任务流程内部、并能把 AI 加到执行权限上的公司：ERP/MES/PLM、工业质检、医疗运营、保险理赔、金融风控、公共部门流程系统、电力调度、智慧物流、智能汽车和机器人。

## A 股风格与行业配置

### 策略判断

我会把本主题定位为**具备可验证流程 ROI 的质量成长**，而不是泛 AI beta 交易。推理成本下降 10x 会扩大可服务市场，但也会压缩通用模型访问的稀缺溢价。这个组合有利于收入可见度更强的应用龙头，不利于 AI 暴露仅停留在 demo、API wrapper 或新闻稿合作的概念股。

### 偏好的配置方向

| 方向 | 观点 | 理由 |
|---|---|---|
| 具备执行权限的垂直软件 | 超配 | 最符合闭环 Alpha：ERP、MES、PLM、医疗 IT、金融 IT、物流、能源管理、税务和政务流程。 |
| Edge AI 和智能终端 | 选择性超配 | 受益于低成本推理和“AI+”终端渗透，但需要验证出货量、搭载率和服务收入。 |
| 工业自动化与机器人软件 | 回调中超配 | 闭环发生在物理世界：感知、决策、控制、反馈。收入兑现可能滞后于叙事，因此估值纪律重要。 |
| 数据、安全与治理基础设施 | 标配至超配 | 是授权数据和 agent 管控的必要条件，但需警惕项目制收入质量。 |
| 纯算力租赁和同质化 IDC | 低配 | 推理成本下降削弱稀缺定价；Jevons effect 可能抬升需求，但利润率面临产能和价格风险。 |
| 薄 AI wrappers 和内容玩具 | 低配 | 切换成本低、数据权弱、流程控制弱，对模型供应商依赖高。 |

### 因子倾斜

更优的因子组合是质量加盈利成长：经常性收入、高毛利、经营现金流为正、R&D 产出效率、客户留存以及低单一模型供应商依赖。AI 应用行情早期，动量因子可能有效；但当降本成为共识之后，市场应更多按 ROI 证据分层定价。

### 估值逻辑

旧的 AI 应用估值经常资本化“模型接入”和总可服务市场叙事。新的估值逻辑应资本化工作流经济性：

| 指标 | 重要原因 |
|---|---|
| AI 收入占经常性软件收入比例 | 区分 demo 与真实付费部署。 |
| 每个自动化任务的毛利 | 把 token 降本与单位经济性相连。 |
| 客户留存和净收入留存 | 检验流程集成是否形成切换成本。 |
| 客户部署回收期 | 衡量真实 ROI 和销售周期摩擦。 |
| 具备执行权限的任务占比 | 区分建议工具和执行系统。 |
| 结果数据飞轮 | 证明使用越多，未来表现越好。 |

只有当至少三个条件可见时，估值扩张才有较强依据：AI 模块已经付费、部署可以复制、客户结果可以衡量。否则，推理成本下降反而可能降低进入门槛并压缩估值。

## 组合结论

结论不是“数据胜过算法”，也不是“agents 胜过数据”。可投资层级应是：

1. **任务执行闭环是核心 Alpha 来源。**
2. **数据孤岛只有在鲜活、授权、并由闭环产生时才重要。**
3. **算法效率只有在闭环内部改善利润率、延迟或产品触达时才重要。**

对 A 股配置而言，这支持对垂直 AI 应用软件和工业/企业流程龙头保持选择性多头倾向，同时回避泛化 AI wrapper 和同质化算力稀缺交易。风格上应保持纪律：买入能展示付费 AI 渗透、可衡量流程 ROI 和数据反馈复利的公司，而不是仅宣布模型接入的公司。

## 关键风险与反证条件

| 风险 | 策略影响 |
|---|---|
| 基础模型通过重大推理突破重新获得稀缺性 | 算法效率和模型所有权重新获得估值权重。 |
| 合规或安全原因导致企业不给 agents 执行权限 | 数据孤岛和治理工具相对跑赢完整自动化公司。 |
| 价格战在流程 ROI 被证明前摧毁软件商业化 | 偏好现金流质量，回避高销售成本应用公司。 |
| 国内算力瓶颈在模型降本后仍然存在 | 端侧优化和国产芯片/软件生态重新获得稀缺溢价。 |
| 政策支持导致主题拥挤 | 用估值和订单质量控制风险，回避纯概念行情。 |
