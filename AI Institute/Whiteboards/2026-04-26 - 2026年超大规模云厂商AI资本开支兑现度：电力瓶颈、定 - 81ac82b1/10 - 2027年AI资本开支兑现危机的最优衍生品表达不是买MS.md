# 港美股与ADR视角下的AI资本开支危机对冲压力测试

## 一、结论先行

我对卡片 9 的方向判断采取 **stress-test**：长AI资本开支受损资产的下行保护、长利率波动、长信用保护三条主线是对的，但原方案在标的定义、融资腿和跨市场覆盖上偏窄。2027年的风险不会只表现为“MSFT/AMZN/NVDA 跌、VIX 升”，而会同时体现为：

1. **美国AI资本开支付款方估值压缩**：MSFT、AMZN、META、ORCL 的自由现金流折现与capex指引可信度被重新定价；
2. **AI资本开支受益方分化**：NVDA 面临GPU份额与客户自研ASIC双重压力，但 AVGO、TSM、台系封测和部分ASIC链条不一定同步受损；
3. **港股与ADR先跌后分化**：HSTECH、KWEB、FXI 会在第一阶段被美元流动性和风险平价减仓拖累，但阿里、腾讯这类现金流/回购驱动资产对“美国电力瓶颈”的基本面暴露低于美股hyperscaler；
4. **港股通资金可能形成反向稳定器**：南向资金可在港股科技回撤时提供价格不敏感买盘，导致用恒生科技或中概ETF来替代美股AI put 的basis risk偏大。

**核心判断**：卡片 9 的“长 put + 卖 SPX call 融资”应改造为 **美国AI capex payer put-spread collar + 半导体相对价值 + 港股/ADR basis 保护 + 信用/利率触发式加仓**。原始 MSFT/AMZN/NVDA 90% put basket 是必要但不充分的尾部保险。

## 二、对卡片 9 的三点压力测试

### 2.1 篮子不是“hyperscaler basket”，而是混合了付款方与受益方

MSFT、AMZN 是AI基础设施付款方；NVDA 是AI资本开支的利润捕获方。把三者等权做 put basket，会在两个情景下产生错配：

| 情景 | MSFT/AMZN | NVDA | 原篮子问题 |
| --- | ---: | ---: | --- |
| 电力瓶颈但AI需求仍强 | capex兑现折损、云收入递延，估值承压 | 近端订单仍强，远端multiple压缩 | NVDA put 不一定同步兑现，拖累组合delta |
| 自研ASIC快速放量 | 推理成本下降，长期capex效率改善 | GPU份额与毛利率预期下修 | MSFT/AMZN put 可能太重，NVDA put 太少 |
| 全面AI capex信心崩塌 | 三者同步下跌 | 三者同步下跌且NVDA beta最高 | 原篮子有效，但不是最便宜表达 |

更合适的篮子应拆成两层：

| 组合层 | 建议标的 | 作用 |
| --- | --- | --- |
| **AI capex payer downside** | MSFT、AMZN、META、ORCL，低权重加入 GOOGL | 捕捉自由现金流、ROIC与capex可信度重估 |
| **AI monetization / GPU premium downside** | NVDA、SMH/SOXX、可选择性加入硬件链高估值标的 | 捕捉GPU份额裂缝、ASIC替代与半导体估值压缩 |
| **ASIC relative hedge** | 相对多 AVGO/TSM、相对空 NVDA/SMH，或用期权表达 | 防止“GPU输、ASIC赢”时裸空半导体篮子失效 |

因此，我不建议把NVDA简单等权放进“hyperscaler put basket”。NVDA 更适合作为单独的高beta尾部腿，或作为 NVDA put spread / AVGO-TSM 相对多头的另一侧。

### 2.2 用 SPX 105% covered call 融资存在错误的风险转移

卡片 9 提到用 SPX 105% covered call 融资AI put成本。这个思路在会计上降低carry，但在风险上有三处缺陷：

1. **卖出的是全市场上行，而买入的是AI局部下行**：若2026下半年非AI板块、金融、能源、医疗带动SPX上涨，SPX call会持续亏损，但AI put尚未兑现；
2. **105%行权价过近**：在2026年仍处“AI繁荣惯性”的背景下，5%上行空间很容易被吃掉，组合会过早变成负gamma；
3. **宏观宽松情景下融资腿最脆弱**：若美联储降息或财政流动性支撑风险资产，SPX上涨与AI电力瓶颈可以同时存在。

更优融资结构：

| 原结构 | 压力测试后建议 | 原因 |
| --- | --- | --- |
| 买 MSFT/AMZN/NVDA 90% put，卖 SPX 105% call | 买 AI payer 90/65 put spread，卖同篮子 115/125 call spread | 融资腿与风险腿同源，避免SPX非AI上涨造成错配 |
| 单一远端2027到期 | 2026Q4、2027Q2、2027Q4三段ladder | capex兑现危机可能通过财报与电力并网节点分段暴露 |
| 裸long vega | put spread + 事件窗口加仓 | 降低远端vol carry，保留尾部convexity |

若必须用指数融资，QQQ 或 XLK call spread 比 SPX 105% call 更干净；但仍不应卖裸call，而应卖价外call spread，避免AI泡沫延长时对冲组合被融资腿反噬。

### 2.3 “港股科技 = 便宜替代put”是危险简化

港股科技与中概ADR可作为流动性风险和全球成长股beta的放大器，但不是美国AI电力瓶颈的纯对冲标的。

| 标的 | 对美国AI电力瓶颈的基本面暴露 | 市场beta | 作为对冲的评价 |
| --- | --- | ---: | --- |
| BABA / 9988.HK | 阿里云与AI投入相关，但美国电力瓶颈直接暴露低 | 中高 | 适合作为风险off辅助put，不适合作为主对冲 |
| TCEHY / 700.HK | AI推理与广告/游戏生态相关，capex纪律较强 | 中 | 下跌初期有beta，后期可能相对抗跌 |
| BIDU / 9888.HK | AI叙事浓，云与自动驾驶投入较重 | 高 | 可作为中概AI beta下行腿 |
| JD / 9618.HK | AI capex相关性低，消费与物流beta更大 | 中 | 不宜纳入AI capex hedge |
| KWEB / HSTECH | 流动性beta强 | 高 | 适合短期风险off，不适合长期基本面对冲 |
| TSM ADR / 2330.TW | 同时受AI总需求与ASIC份额提升影响 | 高 | 应做相对价值，不宜简单做空 |

**压力测试结论**：若2027冲击来自美国电力和公用事业融资，美国AI付款方的基本面受损最直接；港股科技更多是“风险偏好与美元流动性传导”，不是第一性暴露。把 KWEB/HSTECH put 当作便宜替代，会在南向资金逆势买入、人民币政策稳定、港股回购加速时出现对冲失效。

## 三、港股通与ADR的关键basis risk

### 3.1 南向资金是港股科技的非线性变量

HKEX 披露南向交易已成为港股成交的重要组成部分，且2026年3月南向股票与ETF日均成交额明显高于2025年同期。这个结构意味着：当全球资金因AI风险减仓港股科技时，内地资金可能因估值、分红、回购和“资产荒”逻辑反向承接。

对衍生品组合的含义：

- HSTECH put 的delta在美股风险off第一阶段有效；
- 若冲击进入“美国capex可信度”而非“中国互联网基本面”阶段，HSTECH put 的边际beta会下降；
- 港股通不能交易港股期权，南向现货买盘与离岸期权卖方之间存在结构错配，可能压低实际波动兑现。

### 3.2 ADR-H股转换不是免费对冲

以阿里为例，BABA ADR 与 9988.HK 存在可转换关系，但实际交易仍受三层因素影响：

1. **汇率与交易时区**：HKD/USD联系汇率降低长期FX波动，但盘中风险和隔夜跳空仍存在；
2. **流动性池不同**：ADR由美国长线、ETF和期权市场定价，H股受南向资金、港股本地流动性和结构性产品影响；
3. **政策风险不同**：ADR更敏感于美国监管、制裁、审计和地缘政治 headlines；H股更敏感于港股通规则、内地资金偏好和人民币预期。

因此，若组合中使用 BABA/JD/BIDU ADR put，不能假设其自然对冲 9988/9618/9888.HK 现货敞口。需要单独管理 ADR-H股价差、交易时段和汇率basis。

### 3.3 港股衍生品更适合做“区域beta”，不是做“AI电力瓶颈”

可执行工具排序：

| 工具 | 用途 | 局限 |
| --- | --- | --- |
| 恒生科技指数期货/期权 | 对冲港股科技beta与南向波动 | 与美国AI电力瓶颈相关性不稳定 |
| HSI/HSCEI put spread | 对冲港股系统性风险 | AI主题纯度低 |
| KWEB / FXI / MCHI 美股期权 | 美元账户更易执行 | ETF成分与港股通资金行为存在偏差 |
| 单名ADR期权 BABA/BIDU/JD | 交易中概AI/互联网高beta | ADR政策risk会污染AI主题 |
| 700.HK、9988.HK 单名期权 | 更贴近港股本地风险 | 流动性、报价宽度与交易权限限制 |

## 四、改造后的可执行组合

### 4.1 主对冲：美国AI capex payer put-spread collar

**目标**：捕捉2027年capex兑现折损从31%基准上行至40%尾部时，对自由现金流、ROIC和估值倍数的重定价。

| 参数 | 建议 |
| --- | --- |
| 标的 | MSFT、AMZN、META、ORCL 等权或按AI capex强度加权；GOOGL半权 |
| 结构 | 买 2027年12月 90/65 put spread，卖 115/125 call spread |
| 加仓窗口 | 2026Q3-Q4 初建 40%；2027Q1财报后加 30%；若utility OAS>150bps或30y UST>5.30%再加 30% |
| 年化carry目标 | 控制在名义本金的 2.0-3.5%，避免裸long put超过5%的theta负担 |
| 止损/减仓 | 若AI payer 12个月forward FCF预期上修且capex/sales不再上行，减半 |

相比卡片 9 的90%裸put，该结构牺牲了极端跌破65%以下的部分payoff，但显著降低carry，并把融资腿限定在同一风险池。

### 4.2 高beta尾部：NVDA / SMH put spread，而非等权纳入hyperscaler篮子

| 参数 | 建议 |
| --- | --- |
| 标的 | NVDA 单名 + SMH/SOXX ETF |
| 结构 | 2027年12月 85/55 put spread；在波动率回落或财报后分批建仓 |
| 对冲目的 | 交易GPU份额裂缝、毛利率下修和AI供应链去杠杆 |
| 保护措施 | 以小比例多 AVGO/TSM 或买 AVGO/TSM call spread 抵消ASIC替代赢家风险 |

压力测试：若GPU份额被ASIC侵蚀但总AI算力需求仍强，NVDA可能显著跑输AVGO/TSM；若总capex危机爆发，NVDA、SMH、TSM都会跌，但NVDA下跌弹性仍最大。相对价值腿能降低“空错半导体受益者”的风险。

### 4.3 港股/ADR辅助腿：只做流动性冲击，不做核心主题

| 子组合 | 建议结构 | 触发条件 |
| --- | --- | --- |
| 港股科技beta保护 | HSTECH 或 KWEB 80/60 put spread，期限6-12个月滚动 | VIX<18、HSTECH反弹但美股AI信用指标恶化 |
| 中概AI高beta | BIDU/BABA ADR 小比例put spread | 美国科技股风险off叠加中概AI叙事过热 |
| 相对多现金流港股 | 多 700.HK/9988.HK 现货或call spread，对冲空HSTECH | 南向净买入持续、回购收益率上升 |
| ADR-H股basis监控 | BABA ADR/9988.HK、BIDU/9888.HK 价差 | 价差超过转换成本和历史区间时调整 |

港股腿的预算不应超过总对冲premium的20-25%。它是“全球成长股去杠杆”的补充，不应替代美国AI付款方put。

### 4.4 信用/利率触发器用于择时，而不是静态持有

卡片 6/7 已经把信用与利率放大器定义清楚。对港美股衍生品而言，这些指标应成为加仓阈值：

| 触发器 | 阈值 | 操作 |
| --- | ---: | --- |
| BBB utility OAS | >150bps | 增加AI payer put spread 20-30% |
| D/AEP CDS | 单名明显跑宽且NEE/SO稳定 | 建立 D/AEP long protection + NEE/SO short protection |
| 30y UST | >5.30% | 增加payer swaption或降低卖call融资比例 |
| 新发utility 30y NIC | 连续 >20bps | 把对冲期限从6-12个月延长到2027年末 |
| HSTECH vs QQQ相关性 | 20日相关性跌破0.35 | 降低港股beta put权重 |
| 南向净买入 | 科技龙头回撤时连续净买入 | 避免追空港股科技 |

| 情景 | 触发路径 | 美股AI付款方 | NVDA/半导体 | 港股科技/ADR | 最优表达 |
| --- | --- | ---: | ---: | ---: | --- |
| Base | 电力并网推迟、capex效率被质疑，但AI需求仍强 | -10%至-18% | NVDA -15%至-25%，AVGO/TSM相对抗跌 | HSTECH -5%至-12%后分化 | AI payer put spread + NVDA相对空 |

| Tail | D/AEP fallen angel、HY技术冲击、AI订单取消 | -35%至-50% | NVDA -50%+，SMH -35%至-45% | ADR先跌25%至35%，H股受南向承接后相对分化 | 长put spread、VIX call spread、CDX保护、减卖call融资腿 |

## 六、对卡片 9 的最终修正

我支持卡片 9 的大方向，但给出四项修正：

1. **标的修正**：MSFT/AMZN/NVDA 不应称为hyperscaler篮子。应拆成 AI capex payer（MSFT/AMZN/META/ORCL/GOOGL）与 GPU premium（NVDA/SMH）两层；
2. **结构修正**：用 90/65 put spread + 115/125 call spread 取代裸90% put + SPX 105% covered call，减少carry和融资错配；
3. **跨市场修正**：港股/ADR是辅助风险off腿，不是美国AI电力瓶颈的纯替代；南向资金会改变HSTECH与QQQ的相关性；
4. **半导体修正**：ASIC放量不是简单利空所有芯片股。NVDA可能输，AVGO/TSM可能相对赢；半导体对冲必须用相对价值表达。

**一句话结论**：2027年AI资本开支兑现危机可以用衍生品对冲，但最干净的表达不是“买MSFT/AMZN/NVDA put、卖SPX call”，而是“买美国AI付款方下行、单独交易NVDA/GPU份额裂缝、用港股/ADR只补流动性beta，并用utility信用和长端利率作为加仓触发器”。
