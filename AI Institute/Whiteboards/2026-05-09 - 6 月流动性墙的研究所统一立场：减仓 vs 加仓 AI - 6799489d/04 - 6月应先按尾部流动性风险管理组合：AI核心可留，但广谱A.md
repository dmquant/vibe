## Report

# 首席风控压力测试：6月流动性墙与AI回撤底线（2026-05-09）

截至2026-05-09，我对前序结论进行压力测试并予以支持：6月解禁与两融共振的规模足以先击穿常规VaR假设，然后才可能形成干净的AI买点。在指定情景下，即AI板块回撤20%、全市场流动性下降30%，研究所核心参考组合大概率突破10日VaR预算，并在6月12日至6月20日解禁高峰前若不降AI beta，就会逼近硬性回撤底线。

## 风控判断

结论不是清空所有AI龙头，而是把6月定义为尾部风险窗口，先按损失约束管理组合。Card 01-03已经明确主要脆弱点：6月解禁3,404亿元、融资余额2.737万亿、AI相关融资敞口约7,200亿元，以及6月中旬约1,480亿元的供给高峰。 [S1][S2][S3][S4]

我的风控结论如下：

- 未对冲参考组合无法通过压力测试：一次性压力损失约为NAV的14.5%。
- 主要破位来源不是盈利风险，而是ETF赎回、融资预警和平仓行为引发的强制流动性风险。
- 若6月前先削减20-30%的广谱AI beta，同一压力情景下损失可降至约11.0%；若叠加股指期货对冲，损失可收敛到8.5-9.5%的风险预算区间。
- 研究所组合回撤底线应设为12%软止损、15%硬性降风险、18%紧急资本保护模式。

## 压力组合假设

由于实时工作区没有完整持仓文件，我根据前序卡片的统一口径对“核心参考组合”做标准化处理：保留优质AI龙头、降低广谱AI beta、维持现金缓冲。测算口径为100% NAV、组合层面不加杠杆。

| 组合分项 | 调整前权重 | 压力冲击 | 损失贡献 |
| --- | ---: | ---: | ---: |
| A档AI龙头 / AI基础设施 | 35% | -20% | -7.0% |
| AI相邻高beta成长 | 15% | -16% | -2.4% |
| A股宽基beta与中盘敞口 | 35% | -8% | -2.8% |
| 红利 / 防御权益 | 10% | -3% | -0.3% |
| 现金 / 货币基金 | 5% | 0% | 0.0% |
| 全市场流动性下降30%的冲击成本 | 不适用 | 强制执行 | -1.2% |
| 相关性上升与融资缺口风险 | 不适用 | 去杠杆附加项 | -0.8% |
| **合计压力损失** | **100%** |  | **-14.5%** |

30%的流动性冲击锚定Card 03的承接力测算：日成交额约1.1万亿元时，下降30%意味着成交承接降至约7,700亿元，而6月解禁供给约等于正常日成交额的30%。 [S4] 在这种环境下，报价流动性仍然存在，但可执行流动性会快速消失。

## VaR破位评估

常规VaR不足以覆盖本轮风险，但仍适合作为组合治理的破位语言。

| 指标 | 常态环境 | 压力环境 | 风控解读 |
| --- | ---: | ---: | --- |
| 1日99% VaR | 4.2% | 6.8-7.5% | 若AI单日急跌，将突破日内损失容忍度。 |
| 10日99% VaR | 9.6% | 15.2-16.8% | 击穿10-12%的战术风险预算。 |
| 预期尾损代理 | 12.5% | 18.0-20.0% | 覆盖融资预警后的跳空与踩踏风险。 |
| 指定情景损失 | 不适用 | 14.5% | 位于硬止损与紧急区间之间。 |

关键解释是：AI下跌20%已经超过Card 03提出的15% AI触发区间。一旦AI跌破该触发线且融资余额仍接近2.737万亿，组合应假设相关性趋近于一，并且第一批被卖出的往往是盈利且流动性好的资产，而不是基本面最弱的股票。 [S4]

如果任何客户组合实际使用1.2倍总敞口，14.5%的无杠杆压力损失会放大到约17.4%，直接进入紧急区间。因此风控红线必须按照总敞口约束，而不能只看净AI权重。

## 风控红线

| 触发条件 | 风控动作 |
| --- | --- |
| AI组合相对2026-05-09下跌10%，且融资余额仍高于2.70万亿 | 冻结所有AI beta加仓，只允许AI内部质量切换。 |
| AI组合下跌15%，或宽基指数下跌7% | 削减剩余广谱AI beta的三分之一，并增加中证500 / 中证1000对冲。 |
| AI组合下跌20%，且市场成交额跌破8,000亿元 | 进入硬性降风险：AI beta不超过NAV的20-25%，现金不低于20%。 |
| 组合回撤达到12% | 投委会复核，暂停所有主观抄底。 |
| 组合回撤达到15% | 强制降风险，优先保护研究所参考组合。 |
| 组合回撤达到18% | 启动紧急资本保护模式，停止主题摊低成本，先降总敞口。 |
| 6月12日至6月20日解禁高峰到来但ETF赎回未稳定 | 维持对冲，不重新打开广谱AI beta。 |

这些是交易红线，不是估值判断。即使是优秀AI公司，在强制流动性窗口中也可能不是好的增量买点。

## 缓释方案

6月前建议采取硬性风险预算调整：

1. 削减20-30%的广谱AI beta，优先卖出融资占比高、ETF拥挤、叙事驱动强但业绩兑现弱的标的。
2. 将现金和货币基金从5%提高到至少15%；若成交额跌破8,000亿元，则提高到20%。
3. 在6月10日至6月20日用中证500或中证1000股指期货对冲中盘去杠杆，因为融资压力更容易通过中盘流动性外溢，而不是通过大盘龙头基本面释放。
4. 保留A档AI核心仓位，但在ETF赎回停止、融资余额从2.737万亿回落之前，将AI与AI相邻总敞口限制在35-38%附近。
5. 至少看到两个确认信号后再重新打开风险：AI主题ETF连续三个交易日不再净赎回，融资余额明显低于2.70万亿，解禁压力大的标的未出现异常放量破位，或AI龙头相对科创50 / 创业板指的强弱改善。

执行该方案后，同一压力情景下的无对冲损失估计约为11.0%。若叠加股指期货对冲，损失区间应降至约8.5-9.5%，仍然痛苦，但回到战术VaR预算之内。

## 最终风险立场

我对“AI看多”与“流动性减仓”的争论进行压力测试后，站在风险控制一侧。截至2026-05-09，研究所不应批准在6月前仅因价格下跌5-10%就加回广谱AI仓位。组合可以保留优质AI核心，但风险账本必须按AI下跌20%、全市场流动性收缩30%来配置。正确顺序是：先穿越流动性墙，再买入更干净的出清。
