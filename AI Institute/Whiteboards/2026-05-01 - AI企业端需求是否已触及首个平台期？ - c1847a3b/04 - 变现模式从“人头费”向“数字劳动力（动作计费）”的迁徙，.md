# 研究报告：企业软件 Agentic 变现能力与 2H26 续约稳定性分析

## 执行摘要
随着 2H26 续约窗口的临近，企业级 AI 市场正分化为**“高价值动作（Action）”**平台与**“低密度辅助”**工具。从“按席位计费”向“按使用量/Agentic 动作计费”的转型，是填补前几张卡片中提到的 ROI 缺口的核心机制。我们的分析显示，**ServiceNow (NOW)** 和 **Salesforce (CRM)** 凭借其果断转向“基于成效的积分系统”，在锁定专用 GPU 算力续约方面处于最有利地位；而 **Microsoft (MSFT)** 在其核心办公套件中面临局部“利用率缺口”的风险。

## 1. 变现模式重构：从“人头费”到“数字劳动力”
2025–2026 周期见证了 AI 计费方式的根本性转变。这种转变对于证明数百亿美元的 GPU 资本支出（Capex）的合理性至关重要。

*   **Salesforce (Agentforce):** “Flex Credits”模式（每个 Action 0.10 美元）为替代劳动力成本提供了直接路径。通过允许客户将人类席位“置换”为数字 Agent，CRM 确保了即使席位总数下降，营收（及 GPU 利用率）仍能保持稳定或增长。
*   **ServiceNow (Prime Tier):** 其三层自主模型（Foundation, Advanced, Prime）强制企业为“自主专家”付费。这形成了一种“生产化强制机制”，因为只有当 Prime 等级确实替代了 Level 1/2 的人类服务台成本时，该定价才具备 ROI 合理性。
*   **Microsoft (M365 E7 "Frontier"):** 推出 E7 层级（99 美元/用户/月）及 "Agent 365" 治理平台是为了锁住高端 ACV（平均合同价值）。然而，若缺乏横跨所有 Office 应用的统一“按动作计费”表，若“利用率密度”保持低位，MSFT 在 2H26 续约时仍面临席位缩减的风险。

## 2. 赢家与落后者：2H26 续约安全性评估

| 厂商类别 | 2H26 续约稳定性关键机制 | 续约风险画像 |
| :--- | :--- | :--- |
| **运营领军者 (NOW, CRM, ORCL)** | **基于使用量/动作的计费。** 每一个 Agent 循环都触发计费事件，使 GPU 使用量与营收直接挂钩。 | **低风险。** 工作流深度集成带来极高粘性。 |
| **基础设施巨头 (AWS, Azure, GCP)** | **预留实例 (RI) 与专用容量。** 将工作负载转向“托管 Agent”使其能在 Token 之外赚取编排费。 | **中等风险。** 取决于其 SaaS 合作伙伴的成败。 |
| **办公生产力套件 (MSFT, GOOG)** | **高端捆绑 (E7/Workspace AI)。** 依赖治理与安全锁定制止流失。 | **高风险。** 若 Agent 无法证明 20%+ 的时间节省，易受“利用率审计”影响。 |

## 3. 可计量的成本节省：3–7% IT 节省的现实路径
结合卡片 03 的观点，3–7% 的 IT 现金节省可通过以下方式实现：
1.  **L1 服务台自动化：** 特定部门（ServiceNow/Salesforce 领域）可实现 40–60% 的成本降低。
2.  **代码现代化：** 交付速度提升 20–30%（GitHub Copilot/AWS Q），尽管这很难直接转化为净现金节省。
3.  **自动化路由：** 使用低成本模型（如 AWS Nova Micro）进行分诊，将高端 GPU 算力留给深度推理任务。

## 4. 结论：Agentic 层的“生产化强制力”
能够在 2H26 成功续签专用 GPU 容量的厂商，是那些成功将“计费仪表盘”从用户登录转移到 Agent 动作上的厂商。这种转型为 GPU 利用率提供了一个“底价”，而简单的聊天式 AI 无法做到这一点。对于那些无法证明 AI Token 与业务“动作”存在直接关联的通用 SaaS 厂商，我们保持谨慎态度。
