## Summary

2026年3,200亿美元的Capex指引虽由超1.1万亿美元的RPO（剩余履约义务）承保，但存在严重的“承诺-利用率缺口”。这种承保是建立在供应稀缺焦虑下的过度预订。若Agentic层在2H26前无法实现规模化ROI，2023年签署的首批AI大合同将面临重签或延期压力，届时 Capex 存在15-20%的下修风险。关键变量已从“算力霸权”转向“利用率密度”。

## Report

# 研究报告：对 2026 年 3,200 亿美元云巨头 Capex 指引的压力测试

## 执行摘要
美股“四大巨头”（MSFT、AMZN、GOOG、META）2026 年合计约 3,200 亿美元的 Capex 指引目前由超过 1.1 万亿美元的巨额剩余履约义务（RPO）提供背书。然而，结构性的“承诺-利用率（C-U）缺口”正在显现。虽然作为创新前端的公开 API 流量已经走平，但受供应链“FOMO”情绪驱动，企业端承诺容量仍维持高位。这导致 2H26 面临指引下修的显著风险，届时 2023 年签署的首批 3 年期 AI 大合同将迎来 ROI 审计与续约谈判。

## 1. RPO vs. Capex：“虚假”的安全感
3,200 亿美元 Capex 的承保强度在账面上逻辑自洽，但在执行层面高度脆弱。
*   **RPO 统治力**：微软（6,270 亿美元）和谷歌云（4,600 亿美元）的积压订单规模远超其年度 Capex。这为应对短期收入冲击提供了数年的缓冲。
*   **合同刚性**：2026 年的合同已转向“Take-or-Pay”（不使用也需付费）的基础设施模式。与 SaaS 不同，这类合同更难取消，但更容易“延期”，从而导致 Capex 投放推迟。
*   **压力点**：如果已承诺 GPU 集群的利用率持续低于 15-20%，那么“积压订单”在未来续约时将成为负债。

## 2. 公开 API 停滞的“渗透”风险
公开 API 流量是行业的“煤矿里的金丝雀”。
*   **漏斗变窄**：API 使用量的走平表明，新的企业 AI 试点转化为生产环境的速度未达预期（当前转化率仅 25-30%）。
*   **库存积压**：因预期指数级增长而预购容量的企业，现在正坐拥闲置的 GPU 周期。这种“隐形库存”将在 2H26 饱和市场，削弱对新 Capex 的需求。

## 3. 2H26：ROI 的大考
为何 2H26 是关键的转折点：
*   **合同到期**：2023 年是“AI FOMO”之年，首批重大 3 年期承诺在此签署。2H26 标志着第一个显著的续约周期，届时“智能体（Agentic）ROI”必须得到验证。
*   **智能体层测试**：如果到 2026 年中，AI 智能体仍未能实现目标工作流 20-30% 的自动化，那么每年 3,000 亿美元以上的基础设施支出将失去合理性。
*   **下修风险**：我们估计，如果利用率缺口无法弥合，2H26 发生“Capex 重置”的可能性为 15-20%。

## 4. 集中度风险（“锚点租户”的脆弱性）
*   3,200 亿美元的支出高度集中于支持少数超级租户（如 OpenAI、Anthropic）。
*   **系统性风险**：这些“锚点租户”的任何闪失（如融资失败或模型遇到瓶颈）都将立即导致相关云厂商的 RPO 承保失效。

## 结论与战略立场
**立场：压力测试 (Stress-Test)**
虽然由于合同义务，我们认为 2026 年 Capex 不会立即崩溃，但“承保质量”正在恶化。投资者应将监控重点从“积压订单”转向“利用率”和“智能体工作负载密度”。2H26 将是当前 Capex 周期的“真理时刻”。
