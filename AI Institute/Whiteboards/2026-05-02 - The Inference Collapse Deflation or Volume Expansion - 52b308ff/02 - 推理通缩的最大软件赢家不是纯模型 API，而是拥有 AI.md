## 核心观点

我支持 Card 1 的核心判断，并把它转化为可投资的 TMT 技术栈：推理通缩最利好那些能够继续按业务结果收费、同时让每次结果背后的模型成本下降的公司。最强的公司不是简单提供 chatbot，而是同时拥有三件事：

1. **AI-ready data control：** 受治理、带权限、可持续更新的企业或运营数据。
2. **Workflow ownership：** AI 建议转化为已批准动作、工单、代码变更、销售步骤、发票、营销活动或合规记录的系统。
3. **Model-routing leverage：** 能够路由、评估、缓存、观测并替换模型，使更低推理价格更多留在平台侧，而不是一比一让利给客户。

按这个框架，最高确定性的上市公司是 **Microsoft (MSFT)、ServiceNow (NOW)、Salesforce (CRM)、Palantir (PLTR)、Snowflake (SNOW) 和 SAP (SAP)**。最强的后期私有公司是 **Databricks**。**Atlassian (TEAM)、Cloudflare (NET)、Datadog (DDOG)、MongoDB (MDB)、Alphabet (GOOGL) 和 Amazon (AMZN)** 是重要的第二梯队或使能层敞口，但风险结构不同。**NVIDIA (NVDA)** 仍是更便宜推理带来总量扩张的最清晰受益者，但它不是本卡问题的最纯答案，因为它捕获的是算力建设，而不是数据/工作流控制面。

## 可投资技术栈

### 1. Compute And AI Factory Layer

**最佳敞口：** NVIDIA、Broadcom、TSMC、AMD、Arista、hyperscaler capex beneficiaries。  
**Card-02 观点：** 总量维度结构性正面，但不是“谁通过数据和 workflow ownership 受益于推理通缩”的主要答案。

NVIDIA FY2026 Data Center revenue **193.7B 美元** 说明，每 token 成本下降可以与总支出爆发并存。原因很直接：推理从偶尔的人类 prompt，迁移到持续的机器对机器工作。风险在于这一层资本开支重、周期性强，并暴露于客户集中、出口限制、自研芯片和后续利用率消化。它是好的总量层，不是最干净的利润迁移层。

### 2. Hyperscale Cloud And Model Platform Layer

**最佳敞口：** Microsoft、Amazon、Alphabet。  
**Card-02 观点：** Microsoft 最突出，因为它把 cloud 与 workflow surfaces 结合在一起；Amazon 和 Alphabet 是强大的使能层，但不是同等完整的 workflow compounders。

Microsoft 是最佳全栈受益者。它拥有 productivity workflows、identity、security、compliance，通过 GitHub 拥有 developer workflows，通过 Dynamics 拥有 business apps，并通过 Azure AI Foundry 拥有 model routing。公司可以在庞大安装基数上混合 foundation models、small models、retrieval、caching 和 application pricing。FY26 Q3 Microsoft Cloud revenue **54.5B 美元**、Azure 增长 **40%** 支撑需求侧，而 **66%** Microsoft Cloud gross margin 显示 AI 基础设施带来的利润率压力。多头逻辑是：推理通缩会随时间帮助修复这一利润率缺口。

Amazon 和 Alphabet 是高质量 AI infrastructure 与 model-platform 敞口。AWS 季度 segment sales **37.6B 美元**，AI revenue run rate 超过 **15B 美元**；Google Cloud 季度 revenue **20.0B 美元**，backlog 超过 **460B 美元**。两家公司都能路由模型并销售托管 AI platforms。但除非客户在 Bedrock、Vertex AI、Workspace 或 Google/AWS-native data services 上构建业务流程，否则企业 workflow capture 更间接。

### 3. Data Control Plane Layer

**最佳敞口：** Snowflake、Databricks、MongoDB、SAP Business Data Cloud、Salesforce Data 360。  
**Card-02 观点：** 如果平台拥有受治理上下文，这一层就是推理通缩转化为软件利润的核心。

Snowflake 和 Databricks 是最清晰的现代数据平台组合。Snowflake Cortex 的重要性在于，它把 LLM calls 带入受治理的数据边界，并支持多个模型提供商。这意味着客户购买的是受治理的数据操作，而不是松散 API 调用。Databricks 拥有更完整的私有市场 AI infrastructure 叙事：Lakehouse、MLflow heritage、Mosaic AI、AI Gateway、Lakebase、Apps 和 Agent Bricks。在 **4.8B 美元+** revenue run rate 和 **134B 美元** 估值下，Databricks 战略质量很高，但已按共识赢家定价。

MongoDB 是更偏应用的数据敞口。Atlas **29%** 增长显示开发者仍在现代化应用数据基础设施，vector/search/embedding 能力使其与 AI apps 相关。限制是 MongoDB 通常不拥有最终 workflow 或定价包；客户应用才拥有。

### 4. Horizontal Workflow Platforms

**最佳敞口：** ServiceNow、Salesforce、Microsoft、Atlassian。  
**Card-02 观点：** 这是 inference collapse 下最具吸引力的上市软件区域。

ServiceNow 是最清晰的横向 workflow 答案，因为它拥有 IT、employee、customer-service、security、risk 和 operational service workflows。AI Control Tower 与 Microsoft Agent 365 集成直接针对 agent-governance 瓶颈。如果企业部署大量 agents，就需要 permissions、observability、exception handling、escalation 和 auditability。这些是 workflow products，不是 model APIs。

Salesforce 在客户数据和面向客户的 workflows 中同样强势。Agentforce and Data 360 ARR 超过 **2.9B 美元**，是 agentic software 正在形成真实收入线的最清晰披露之一。Salesforce 也有可信的 outcome-style economics 路径，因为 sales/service agents 可以绑定 lead conversion、call deflection、case resolution 和 campaign performance。

Atlassian 是第二梯队 workflow compounder。Jira 和 Confluence 存储软件和团队执行的丰富组织上下文；Rovo usage **20%+ month-over-month** 增长显示早期 traction。开放问题是 TEAM 能否足够快地把 AI 定价转化为更高 ARPU 和 enterprise expansion，还是 AI 最终只是 bundled retention feature。

### 5. Vertical And Ontology Workflow Layer

**最佳敞口：** Palantir、SAP，以及拥有专有记录和受监管 workflows 的 vertical software vendors。  
**Card-02 观点：** 这一层可能产生最高 ROI 证明点，因为数据混乱、权限具体、输出直接绑定决策。

Palantir 是最高 beta 的上市案例。FY2025 **56%** revenue growth、**109%** U.S. commercial growth 和 **51%** adjusted free cash flow margin 解释了为什么市场重视它：AIP 正在把碎片化企业数据转化为运营决策并实现货币化。风险是估值和客户集中敏感性，但结构上它符合 inference-collapse winner profile。

SAP 增速较慢，但战略地位强。ERP 是典型 workflow system of record：finance、procurement、supply chain、HR、spend 和 manufacturing。更便宜的推理会让 Joule 和 Business Data Cloud 在核心套件中更有用。SAP 的优势是深度 workflow gravity；限制是产品周期观感较慢，且 model-routing 披露不如 Snowflake 或 Databricks 明确。

### 6. AI Traffic, Observability, And Security Layer

**最佳敞口：** Cloudflare、Datadog、部分 security platforms。  
**Card-02 观点：** 当 agentic systems 变得分布式、可观测且安全敏感时，这一层受益。

Cloudflare 的 AI Gateway、Workers、network 和 security layer 与 AI calls 增多、模型提供商切换的世界高度一致。其 Q4 revenue **34%** 增长和 FY2025 **30%** 增长支持需求，但 workflow control 弱于 NOW/CRM/SAP。

Datadog 是“AI systems must be operated”的强敞口。当应用调用多个模型、tools、vector stores、databases 和 cloud services 时，企业需要 logs、traces、LLM observability、cost monitoring、security telemetry 和 incident response。Datadog 不拥有业务 workflow，但越来越拥有 AI-native software 的 operating workflow。

| 核心 control-plane compounders | MSFT, NOW, CRM, PLTR, SNOW, SAP | 数据控制、workflow ownership 和保留模型成本下降收益的能力最佳。 |
| 需要跟踪的私有赢家 | Databricks | 最强后期私有 data/AI control-plane asset；估值是主要约束。 |
| 使能型云平台 | AMZN, GOOGL | AI 需求捕获巨大，但企业 workflow economics 不如 MSFT/NOW/CRM 直接。 |
| AI operations and routing layer | NET, DDOG, MDB | 受益于模型扩散、routing、observability 和 app-data modernization。 |
| Compute volume layer | NVDA, AVGO, TSM, AMD, ANET | 受益于总推理扩张；更暴露于 capex-cycle 和 utilization 风险。 |
| 观察名单，不是本筛选顶层 | ADBE, INTU, WDAY, HUBS, Figma, Canva, Notion | workflow/data surfaces 强，但 model-routing leverage 或企业 action ownership 的当前证据不如顶层公司完整。 |

## 应避免的方向

1. **没有专有 workflow 的纯 model API resale。** 如果供应商不能把模型成本隐藏、路由或优化进业务结果，token 价格下降会压缩 gross profit。
2. **数据权利薄弱的 prompt-wrapper software。** 这类公司可能有 usage，但缺少持久定价权。
3. **没有治理和语义层的数据平台。** 原始存储不是 AI-ready data control。
4. **仅作为 retention subsidy 的 AI features。** 如果 AI 只是防 churn 的捆绑功能，而不能通过 ARPU 货币化，usage growth 未必创造股东价值。
5. **按无限 capex growth 定价的 infrastructure stories。** 总量扩张是真实的，但若 utilization、电力或客户集中度不及预期，算力层可能过冲。

## 结论

推理通缩 regime 奖励那些能把更便宜模型调用转化为更高利润工作的公司。因此，最好的可投资技术栈不是简单“买所有 AI 算力”。更合适的是 barbell：选择性持有 compute 和 cloud volume winners，同时超配最靠近受治理数据和可执行工作流的软件控制面。

我对本卡问题的最高确定性答案是：

**上市：** MSFT, NOW, CRM, PLTR, SNOW, SAP。  
**私有：** Databricks。  
**第二梯队：** TEAM, NET, DDOG, MDB, GOOGL, AMZN。  

如果推理价格继续下降，这些公司能够把任务路由到更便宜的模型，扩大 AI-assisted tasks 的数量，并通过 seats、workflows、outcomes、governance 和 data products 保留更多经济性。
