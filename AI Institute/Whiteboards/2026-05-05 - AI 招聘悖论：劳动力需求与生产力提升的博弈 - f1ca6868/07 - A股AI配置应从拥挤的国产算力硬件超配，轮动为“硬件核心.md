## Report

# A股AI全链条配置框架：从算力硬件到应用端
**工作日期：2026-05-08 | 第7/8张卡 | 分析师：首席策略师 [chief-strategist] | 立场：综合**  
**白板：f1ca6868-bfb3-4c38-aa01-2adce80fc48d**

截至2026-05-08，我对前六张卡的判断是综合而非否定：AI招聘悖论是企业落地阶段的实施成本，长期生产力红利仍然成立，中国AI周期呈双速结构，但A股硬件交易已经反映了过多利好。组合答案不是退出AI，而是从“国产算力单腿高beta”转向攻守兼备的杠铃：保留硬件核心仓位，补足低拥挤度应用端，并配置足够的价值与现金缓冲，以应对Q2-Q3宏观数据扰动和杠杆清洗。

## 核心判断

未来6-9个月，A股AI配置应采用**防守型成长轮动**：

- **适度减持算力硬件**：国产替代逻辑仍在，但在TMT成交额占比接近38.4%、主动基金TMT配置达28.6%、龙头芯片融资余额占流通市值8.5%-10.2%的背景下，不应继续加杠杆追高。
- **向AI应用端补仓**：电力IT和智能驾驶应吸收增量资金，因为二者机构拥挤度更低、业绩催化更近，商业模式与真实部署挂钩，而不只是资本开支预期。
- **用价值因子对冲成长因子**：组合仍应由成长主导，但必须搭配高股息电力/电网运营商、电信运营商、银行以及现金或短久期工具。
- **执行硬风控触发器**：若TMT成交额占比超过45%、硬件融资余额周环比下降超过3%，或硬件龙头放量单日下跌5%-8%，应先降低beta，再讨论基本面。

## 1. 前六张卡已经证明什么

| 逻辑链条 | 投资含义 |
|---|---|
| 第1张卡：AI招聘增加，是因为企业先需要实施型劳动，之后才可能减少净人力。 | AI部署真实存在，但短期ROI会被数据、集成、安全和流程成本“征税”。 |
| 第2张卡：部署支出最先流向云和网络安全，数据基础设施滞后2-4个季度。 | 最先兑现收入的是基础设施和安全，而不是广义应用软件。 |
| 第3张卡：AI周期的宏观生产力红利通常滞后资本开支8-16个季度，约在2027-2029年可见。 | 不必等2026年宏观TFP确认才持有AI，但也不能为它支付过高价格。 |
| 第4张卡：中国AI呈双速结构：平台场景可在2026-2028年率先体现生产力，长尾B端更慢。 | A股应偏向国产算力、电力IT、智能驾驶和产业部署；平台变现更多体现在港股互联网。 |
| 第5张卡：国产算力硬件拥挤，TMT成交额占比38.4%，主动基金TMT配置28.6%，芯片融资余额占流通市值8.5%-10.2%。 | 硬件应从扩张仓位降为核心仓位，上行不再是干净beta。 |
| 第6张卡：智能驾驶和电力IT拥挤度更低，智能驾驶专项配置约4%-6%，ADAS融资余额3%-5%，电力IT融资余额1%-3%。 | 应用端轮动的12-18个月风险收益比优于继续追硬件。 |

因此，综合结论很明确：**AI仍是战略主题，但A股主线应从资本开支稀缺转向部署变现。**

## 2. 配置框架

建议把AI配置拆成两层：总权益组合中的AI预算，以及AI主题内部结构。

### 2.1 总权益组合预算

对于均衡型A股权益组合，AI相关敞口应维持在权益资产的**35%-45%**，在硬件杠杆仍高时不应超过50%。内部结构如下：

| 仓位桶 | 占总权益组合建议权重 | 角色 |
|---|---:|---|
| 国产算力硬件与AI服务器 | 8%-12% | 战略核心，但从激进超配下调 |
| 智能驾驶与E2E ADAS | 10%-15% | 带有软件收入期权的应用端成长beta |
| 电力IT与电网数字化 | 8%-12% | 有政策支撑的防御型AI应用 |
| 安全、数据基础设施、产业/企业软件 | 4%-6% | 部署层受益，但A股纯度较弱 |
| 价值缓冲与流动性储备 | 15%-20% | 控制回撤并保留轮动弹药 |

剩余权益仓位可配置非AI周期/价值、消费、金融、公用事业或其他机构观点。关键约束是：AI敞口必须在**算力、部署、应用**之间分散，而不是集中在芯片。

### 2.2 AI主题内部结构

若以100%的AI主题组合衡量：

| 主题 | 主题内权重 | 操作姿态 |
|---|---:|---|
| 国产芯片 / AI服务器 | 25%-30% | 持有核心龙头；急涨减仓；不加融资杠杆 |
| 智能驾驶 | 25%-30% | 板块回调时积累；偏好软件/royalty敞口 |
| 电力IT | 20%-25% | 稳步建仓；作为AI内部防御锚 |
| 网络安全 / 数据 / 企业软件 | 10%-15% | 选择性持有部署受益者 |
| 主题内现金 / 对冲 | 10%-15% | 为杠杆清洗后的再配置保留空间 |

这是一条明确的轮动路径：硬件不应清零，因为国产替代周期真实存在；但在第5张卡确认估值和杠杆压力后，它不应继续主导AI组合。

## 3. 行业判断

### 3.1 国产算力硬件：保留核心，停止追高

长期逻辑仍然成立：出口管制和本土模型部署将持续支撑寒武纪(688256.SH)、海光信息(688041.SH)、工业富联(601138.SH)、浪潮信息(000977.SZ)等公司的需求。但交易已经从“盈利上修叠加估值重估”变成“盈利必须验证估值”。

第5张卡给出的边界必须被纳入组合纪律：

- 国产芯片龙头约为**18x-22x 2026E P/S**，历史极值锚约**25x P/S**。
- AI服务器龙头约为**18x-22x 2026E P/E**，接近**25x P/E**的周期顶部锚。
- 硬件融资余额已经高到足以使**5%-8%单日回调**演变为被动卖出风险。

策略：总权益组合保留8%-12%的硬件仓位，但逢高再平衡。TMT成交额占比高于40%时不再加仓；若突破45%，应降低风险。

### 3.2 电力IT：AI组合的防御锚

电力IT是最清晰的“带价值属性的AI应用”。它有两个相互独立的需求来源：

- AI数据中心带来高负荷、高波动的电网需求，需要更好的负荷预测、调度和需求响应系统。
- 中国新型电力系统建设与新能源并网要求电网数字化，无论AI行情强弱都要推进。

优选标的包括国电南瑞(600406.SH)、朗新科技(300682.SZ)、南网科技(688248.SH)、思源电气(002028.SZ)、许继电气(000400.SZ)。这个仓位桶的任务不是最大化beta，而是在硬件去杠杆时保持AI组合仍然可投。

策略：总权益组合建至8%-12%，或AI主题内20%-25%。若全TMT回撤但电网订单、国家电网/南方电网资本开支信号未变，应逆势加仓。

### 3.3 智能驾驶：有催化风险的应用成长

智能驾驶是上行弹性最高的应用端仓位。端到端模型、中国NEV数据飞轮，以及L2++渗透率从**2025年约15%**向**2026E的28%-32%**提升，构成了软件变现路径。主要上市载体包括德赛西威(002920.SZ)、Horizon Robotics(9660.HK)、BYD(002594.SZ / 1211.HK)、赛力斯(601127.SH)、XPeng(9868.HK)、均胜电子(600699.SH)。

风险在于时间：如果MIIT L3商业化框架晚于2026H2落地，付费软件收入可能推迟6-12个月。因此，这一仓位应在回调中积累，而不是在消息刺激后追涨。

策略：总权益组合建至10%-15%，或AI主题内25%-30%。优先选择royalty、域控制器和软件赋能供应商，而不是纯硬件BOM供应商。

### 3.4 安全、数据基础设施和企业软件：选择性配置，不做一揽子买入

第2张卡显示，全球维度下网络安全是AI部署最早、质量最高的受益者之一。但在A股，直接纯度和公司质量更不均衡。深信服(300454.SZ)、奇安信(688561.SH)、绿盟科技(300369.SZ)、Kingdee(0268.HK)、用友网络(600588.SH)值得跟踪，但仓位应反映较慢的变现、采购周期和利润率波动。

策略：总权益组合配置4%-6%，或AI主题内10%-15%。这是验证型篮子，而不是主要风险来源。

## 4. 成长与价值因子的协调

正确的风格立场是**成长为主、价值对冲**，而不是成长和价值之间二选一。

| 宏观/市场状态 | 成长敞口 | 价值敞口 | 操作 |
|---|---:|---:|---|
| 基准情形：增长温和，4月PMI稳定，政策仍支持 | AI相关主动风险的60%-65% | 35%-40% | 持有应用成长，同时配置电力IT和股息缓冲 |
| 宏观上行超预期：5-6月数据再加速，周期股扩散 | 50%-55% | 45%-50% | 先减拥挤硬件；增配银行、电信、高股息公用事业和部分周期 |
| 宏观下行超预期：需求走弱，风险偏好下降 | 55%-60% | 40%-45% | 保留电力IT；降低高P/S硬件和未盈利软件 |
| 杠杆清洗：TMT成交额占比>45%或硬件融资余额周降>3% | 40%-50% | 50%-60%并提高现金 | 先降beta，再在被动抛售中择机回补 |

2026年4月官方PMI支持均衡配置，而不是纯risk-on：制造业PMI为**50.3%**，较3月下降**0.1个百分点**但仍在扩张；非制造业商务活动指数为**49.4%**，下降**0.7个百分点**；综合PMI产出指数为**50.1%**。一季度GDP同比增长**5.0%**，一季度规上工业企业利润同比增长**15.5%**。这组数据足以避免衰退定价，但结构不均衡，足以惩罚高beta拥挤交易。

## 5. Q2-Q3风险纪律

### 5.1 必须降风险的触发器

| 触发器 | 阈值 | 组合反应 |
|---|---:|---|
| TMT成交额占比 | >40%警惕；>45%退出信号 | 停止加仓硬件；降低高beta芯片和服务器 |
| 硬件融资余额 | >10%流通市值或周环比下降>3% | 视为杠杆清洗信号；提高现金 |
| 单票回撤 | 放量单日下跌5%-8% | 先降低beta，再评估基本面 |
| 解禁 / 大宗交易 | Q2-Q3 2026硬件翻倍股集中出现 | 供给事件前预先减仓 |
| 毛利率低于预期 | 2026H1硬件中报不及预期 | 更快从P/S切到PEG估值 |
| 宏观上行超预期 | 5-6月数据强且周期扩散 | 从硬件成长转向价值/周期对冲 |
| 宏观下行超预期 | 需求和服务同时走弱 | 保留电力IT；降低长久期软件和高P/S标的 |

### 5.2 再进入规则

不要买硬件的第一根下跌。只有至少两个条件出现后，才重建仓位：

- TMT成交额占比回落至**32%-35%**以下。
- 硬件融资余额连续两周下降，且未造成宽基指数压力。
- 2026E EPS或收入指引被上调，而不是仅被维持。
- 抛售主要来自仓位因素，而非订单取消或毛利率恶化。
- 应用端龙头在硬件调整期间保持相对强势。

## 6. 最终组合观点

七张卡串联后的可投资结论是：

1. **支持AI结构性主线**：招聘悖论是部署阶段，不是AI失效证据。
2. **支持中国双速结构**：平台和高密度产业场景会早于长尾B端体现生产力。
3. **压力测试A股硬件交易**：拥挤和杠杆限制短期上行。
4. **综合为轮动策略**：把硬件从主导性超配降为核心仓位，把智能驾驶和电力IT作为主要增量配置。
5. **用因子纪律管理组合**：保持成长主线，但在杠杆清洗风险结束前，用价值、股息和现金进行对冲。

对2026年Q2-Q3而言，组合应当做到：**持有AI，但持有那些能穿越硬件调整的AI资产。**
