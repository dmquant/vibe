## Summary

API 流量增速首次走平更像是企业 AI 从实验扩张进入 ROI 过滤阶段，而不是需求全面见顶。调研证据显示采用面很广但财务影响有限：McKinsey 称 88% 组织常规使用 AI，但只有约三分之一开始规模化；Gartner 称至少 50% GenAI 项目在 PoC 后被放弃。与此同时，Microsoft、Alphabet、Amazon 和 NVIDIA 的最新数据仍显示云 AI、backlog、AWS、Google Cloud、Azure 和数据中心芯片需求强劲。结论是：浅层试点和薄封装工具承压，生产性工作负载、云平台和算力链仍有支撑。

## Report

# 企业 AI 需求是否触及首个平台期？

**日期：** 2026-05-02 SGT  
**分析师：** TMT Analyst  
**立场：** 初始判断，偏向压力测试

## 核心判断

API 流量增速首次走平值得重视，但还不足以判断企业 AI 需求已经进入持续性平台期。我的基准判断是，市场正在从“实验驱动的放量阶段”切换到“ROI 筛选后的生产阶段”。这个切换会让通用试点、薄封装 AI 工具、以及无法证明降本或增收的工作负载承压，但还不意味着云 AI 基础设施、半导体或深度嵌入企业流程的 AI 需求全面衰退。

最准确的表述是：**实验阶段正在结束，但需求没有结束；需求正在被重新定价、纳入治理，并向少数高确定性场景集中。**

## API 平台期大概率意味着什么

本轮 API 流量走平是一个有用的早期信号，因为 API 能最快反映开发者边际实验行为。当一个团队不再同时测试五个聊天机器人、十个 copilot 和多条 prompt chain 时，请求量增速可能会先于预算削减而走平。它也最容易受到 CFO 行为影响，因为用量账单容易审查、限流和重新分配。

但 API 调用数并不是完整的需求代理指标。OpenRouter 的 100 trillion token 研究之所以重要，是因为它显示真实使用正在转向更重、更依赖上下文、更多 reasoning 和编程类工作负载，而不是简单的一次性 prompt。自 2024 年初以来，平均 prompt token 长度接近增长四倍，completion 长度接近增长三倍；在有标签的数据中，编程是 prompt-token 增长的最大驱动因素。[^1] 因此，请求数走平可以与更高 token 强度、更高延迟要求和更高基础设施负载同时存在。

这里还存在渠道结构问题。OpenRouter 自身也提示，其数据来自单一平台，无法完整覆盖企业使用、本地部署或封闭的内部系统。[^1] 随着企业从实验走向生产，更多用量可能从可见的第三方 API 聚合平台消失，转而出现在 Azure、AWS Bedrock、Google Cloud、私有部署或应用原生 AI 功能中。

## 支持 ROI 审计阶段的证据

“ROI 审计”判断最强的证据来自企业调研，而不只是流量数据。

| 证据 | 对需求的含义 |
|---|---|
| McKinsey 2025 年调研显示，88% 的组织在至少一个职能中经常使用 AI，但只有约三分之一开始在全组织范围扩展 AI 项目；39% 报告 AI 对企业层面 EBIT 有任何影响，其中多数影响低于 EBIT 的 5%。[^2] | AI 采用面很广，但可规模化的财务影响仍少。此时 CFO 审查自然会加强。 |
| Deloitte 2026 年企业 AI 报告显示，66% 的组织报告生产率或效率改善，但目前只有 20% 报告 AI 带来收入增长，另有 74% 希望未来通过 AI 增收。[^3] | 收益结构仍偏向效率，而不是收入端转型。收入导向的 AI 预算需要更强证明。 |
| Gartner 表示，截至 2025 年底，至少 50% 的 GenAI 项目在概念验证后被放弃，原因包括数据质量差、风险控制不足、成本上升或商业价值不清晰。[^4] | 审计已经在项目层面发生。 |
| Slalom 2026 年执行摘要显示，91% 计划在 2026 年增加 AI 投资，但只有 39% 跟踪 ROI；只有 21% 的用例实现企业级扩展，28% 嵌入端到端流程。[^5] | 预算仍可增长，同时低质量试点会被清理。约束不只是热情下降，而是运营纪律上升。 |

这些证据支持一个更有选择性的下一阶段。“AI 预算”不再等于所有 proof of concept 都能获得通行证。证明责任转向可度量的流程重构、cost-to-serve 下降、开发者生产率、客服自动化、反欺诈和风控运营、软件工程以及垂直知识工作。

## 反驳需求崩塌的证据

截至最近一个财报季，上市公司证据并不像企业 AI 需求已经全面掉头。

Microsoft FY26 Q3 收入为 $82.9 billion，同比增长 18%；Microsoft Cloud 收入为 $54.5 billion，同比增长 29%；Azure and other cloud services 收入增长 40%；commercial remaining performance obligation 增长 99% 至 $627 billion；AI business 年化收入 run rate 超过 $37 billion，同比增长 123%。[^6]

Alphabet Q1 2026 收入为 $109.896 billion，Google Cloud 收入为 $20.028 billion，去年同期为 $12.260 billion；Google Cloud 经营利润从 $2.177 billion 升至 $6.598 billion。[^7] 这与“企业 AI 需求已经停滞”的简单叙事不一致。

Amazon Q1 2026 AWS 销售额为 $37.6 billion，同比增长 28%，是 15 个季度以来最快增速；AWS 经营利润为 $14.2 billion。Amazon 同时披露，过去 12 个月自由现金流从 $25.9 billion 降至 $1.2 billion，主要因为 property and equipment 采购增加 $59.3 billion，反映 AI 投资。公司还提到芯片业务年化收入 run rate 超过 $20 billion，并披露 OpenAI 和 Anthropic 的重大 AI 基础设施承诺。[^8]

Meta 是另一类信号：它不是云服务卖方，但 Q1 2026 收入增长 33% 至 $56.311 billion，同时继续围绕 AI 和 superintelligence 投资讲述业务方向。[^9] 这强化了资本开支强度与货币化时点的争论，尽管它不是第三方企业需求的直接证据。

在半导体层面，NVIDIA FY26 Q4 数据中心收入达到 $62.3 billion，环比增长 22%、同比增长 75%，全年收入为 $215.9 billion。[^10] 算力供应链仍体现出远超普通软件周期放缓的基础设施需求。

## 我的解释框架

表面矛盾可以通过拆分三层需求来解释：

1. **探索性流量：** 这层最可能先平台化。它包括沙盒 prompt、缺乏纪律的 API 测试、重复试点，以及没有业务流程 owner 的团队用量。这一层最脆弱。

2. **生产性 inference：** 这层仍在增长，但很难通过原始 API 调用数观察。生产用量通常会被批处理、嵌入应用、纳入治理、跨模型路由、缓存，并对内部 KPI 负责。它更可能体现在云收入、RPO、token 强度、GPU 利用率和应用毛利率中，而不是公开 API 请求数中。

3. **AI 基础设施建设：** 这层仍然强劲，但现在面临更高审查，因为部分产业链的资本开支增长快于自由现金流。关键问题不再是“有没有人在用 AI”，而是“哪些工作负载能够吸收折旧、电力、内存、网络和 inference 成本”。

所以，是的，API 平台期大概率标志着最容易的实验阶段结束。不是的，它还不代表企业 AI 需求结束。

## 谁最先受伤

最先承压的应是：

- 只销售广义生产率愿景、但缺乏可审计用量到节省转换逻辑的 AI SaaS 工具。
- 缺乏专有流程数据、分发能力或切换成本的 model wrapper。
- 依赖脆弱 orchestration、但没有治理、回滚机制和清晰任务完成指标的 agentic 项目。
- 业务 owner 无法说清 AI 前基准、目标 KPI 和回收期的内部试点。
- 在低风险任务中可被更便宜 open-weight 或小模型替代的 frontier API 用量。

## 谁仍可能复利增长

受益者应是：

- 拥有强企业合同、模型选择、合规控制和数据重力的云平台。
- 受益于持续 inference 和训练容量的半导体、网络、内存、冷却和电力供应商，不过估值敏感性正在上升。
- 将 AI 嵌入系统记录并能披露 attach、留存或 seat expansion 的企业软件厂商。
- AI 能够可量化降低 cost-to-serve 或缩短周期的垂直流程，例如软件工程、客服、欺诈审核、理赔、财务运营、采购和知识管理。

## 下一步观察指标

如果以下多项同时出现，平台期论点会明显转熊：

- API 请求数走平，同时每请求 tokens、completion 长度和付费转化也走平。
- hyperscaler 云增长连续两个季度放缓，且管理层无法用产能解释。
- commercial RPO/backlog 增长放缓，转化周期拉长。
- 资本开支指引下调的原因是客户需求弱化，而不是供应或组件价格变化。
- AI 应用厂商报告 2024-2025 年采用 cohort 的净留存下降或扩张变弱。
- GPU 利用率、租赁价格和交付周期同时走软。

如果出现以下情况，平台期论点会被削弱：

- 云 AI 收入和 backlog 继续加速。
- 企业削减试点，但扩大生产工作负载。
- 即使请求数走平，每请求 tokens 和 inference 算力强度仍继续上升。
- AI 使用从独立 API 调用转向应用原生或私有部署。

## 结论

市场大概率已经过了“到处试 AI”的阶段。这是一个真实的阶段变化。但更准确的说法是：这是第一轮 ROI 过滤，而不是第一个企业 AI 需求平台期。下一阶段会奖励能证明流程级经济性的厂商，并惩罚那些把实验噪音货币化的公司。

对投资者而言，风险不只是 AI 需求消失。更尖锐的风险是**结构变化**：资金从浅层实验迁移到更少、更重、更集成、更可衡量的工作负载。这可以继续支撑 hyperscale cloud、半导体和嵌入式企业平台，同时压缩通用 AI 软件和 API-only 增长故事的估值。
