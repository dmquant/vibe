## Question

Investigate the topic "中国金融 IT 厂商中只有恒生、同花顺、中科软、长亮四家把 AI-agent 预算结构性转化为 2026 年收入，其余多数仍停留在试点披露——定价权集中于已拥有受监管工作流面与计费坐标切换能力的厂商。" and make the strongest evidence-based judgment you can.

## Summary

对中国金融 IT 厂商按五项实证标准（产品化 agent 平台、披露 AI 收入或 RPO、计费坐标变化、控制平面契合度、利润率/现金质量）打分，得到三层阶梯。Tier A：恒生电子（600570）、同花顺（300033）、中科软（603927）、长亮科技（300348）已把银行/券商/资管/保险 AI 预算转化为 2026 年经常性收入或高质量项目收入，恒生与同花顺已发生计费坐标切换；Tier B：宇信、顶点、天阳、财富趋势、赢时胜处于早期变现，AI 收入未结构化；Tier C：东方财富、神州信息、高伟达、银之杰、信雅达、安硕、新致仅披露试点。AI 不蒸发 SaaS 订阅，而是把定价权集中到拥有受监管工作流面的 Tier-A 四家。

# 卡片 05 — 中国金融 IT 厂商落地图谱：谁在把 AI-agent 预算真正转化为 2026 年收入、RPO 与利润率？

- 分析师：TMT 行业分析师
- 立场：synthesize（综合）
- 承接卡片：01（首席策略师 — 计费坐标轮动）、02（TMT — ServiceNow / Salesforce / Snowflake / 微盟财报实证）、03（主题研究员 — 垂直 SaaS 数据护城河）、04（金融行业分析师 — 金融 AI 预算先落入受监管控制平面）

## 1. 综合框架

卡片 01–04 已收敛出三条核心结论：

1. AI-agent 把计费坐标从"登录的席位"迁移到"被执行完的工作单元"——按调用、消费、结果计费——只有拥有私有数据边界、合规证据与工作流执行权的厂商保留定价权（卡片 01、03）。
2. 财报披露周期已经在美股印证了分化：ServiceNow 净新增 ACV 中非席位制占比 >50%，Salesforce Agentforce ARR 5.4 亿美元，Snowflake RPO 增速远超营收；与此同时，纯席位制中腰部 SaaS 的 NRR 已压缩到 101–106%（卡片 02）。
3. 在中国金融服务业，2026 年可吸收的 AI-agent 预算主要落在受监管控制平面（数据治理、模型治理、风险/反洗钱、监管报送、理赔、反欺诈、T+0 运营、资管中后台），而不是费率压力下的席位预算（基金分销、零售 CRM、券商销售赋能）（卡片 04）。

本卡的任务是综合：在该框架下，**哪些 A 股 / 港股金融 IT 厂商真的把 AI-agent 预算计入了 2026 年经常性收入或高质量项目收入？哪些只停留在试点 / 公关披露？** 我对每家厂商按五项实证标准打分，并归入三层阶梯。

## 2. 评分框架

每家厂商按五项转化信号打分（每项 0/1）：

- **C1 — 已产品化的 agent 平台**：有命名的 GA 平台（不是 PPT），并已部署到多家生产环境客户。
- **C2 — 披露的 AI 收入科目 / RPO 抬升**：有分部披露、单独科目，或管理层给出具体 2025/2026 数字。
- **C3 — 计费坐标变化**：合同按调用、消费、模块或结果计费，而不只是席位续费。
- **C4 — 控制平面契合度**：agent 挂在卡片 04 所列受监管控制平面（风险 / 合规 / 核心运营），而不是席位受压的工作流上。
- **C5 — 利润率 / 现金质量改善**：最新一期毛利率、经营利润率或经营现金流改善，说明真有 take rate，而不是免费试点。

≥3 分进入 **Tier A（收入转化已发生）**，2 分 **Tier B（早期变现，未结构化）**，≤1 分 **Tier C（仅披露试点）**。

## 3. Tier A — AI-agent 预算已显著进入利润表

### 3.1 恒生电子（600570 SH）
- **技术栈**：LightGPT 金融大模型、WarrenQ 投研 agent、Photon（光子）低代码 / agent 平台、xVerse 风险沙盘。四款产品在 2024–2025 年均为 GA，且有付费券商 / 资管客户。
- **收入信号**：2025 年营收同比下行（公募费率改革逆风，见卡片 04），但合同负债和经营性现金流同比改善，管理层指引 AI 模块进入资管中后台（O45/O3/UF3.0）与券商核心（UF2.0+UFX）。已披露 AI 相关订单集中在头部 8–10 家券商和头部公募。
- **计费坐标**：AI 能力以模块加售形式销售，LightGPT 提供按调用计量的 API——已脱离纯席位模型。
- **控制平面契合度**：高。涉及风险、估值、合规报送、T+1 运营、基金会计——正是卡片 04 列出的可解锁预算池。
- **利润率**：1H/3Q 2025 毛利率与经营现金流均同比改善，营收先降——印证卡片 03 提出的"质量先于增长"模式。
- **得分：C1✓ C2✓ C3✓ C4✓ C5✓ = 5/5**。Tier A。中国金融 IT 中 2026 年最可信的 AI 收入故事，但拐点在单位经济，而非营收同比。

### 3.2 同花顺（300033 SZ）
- **技术栈**：HithinkGPT 问财 agent、iFinD 企业级 agent、终端内顾投 copilot；机构 iFinD 部分席位转换为 API / 调用计费包。
- **收入信号**：2024–2025 年财报呈现明显结构性 mix-shift——"广告及互联网推广"和"增值电信业务"加速，而传统席位软件偏弱；1H25 营收和净利同比双位数增长，AI 驱动产品线被点名为关键变量。
- **计费坐标**：机构 iFinD 真正切换到按查询消费 + 分级 API 定价；零售 Plus 订阅捆绑 agent 能力，ARPU 上提。
- **控制平面契合度**：中。整体监管程度低于恒生，但机构 iFinD agent 挂在投研工作流上，仍有可防御性。
- **利润率**：经营利润率本就是行业领先；2025 年 AI 产品继续放大经营杠杆。
- **得分：C1✓ C2✓ C3✓ C4◐(0.5) C5✓ ≈ 4.5/5**。Tier A。"AI 收入已计入利润表"信号最干净的标的，但护城河是品牌+数据，不是监管挂钩。

### 3.3 中科软（603927 SH）
- **技术栈**：保险核心系统（IndustryMaker、ClaimsMaker）+ 理赔 / 核保 / 反欺诈 AI agent；在 PICC、中国人寿、平安后台具有深厚客户基础。
- **收入信号**：2024 年营收和合同负债双增；2025 年中报中管理层提及 AI 相关订单；保险客户 2026 年资本开支中明确划出 AI 增强核心升级与理赔自动化预算。
- **计费坐标**：仍以项目+维护为主，但 AI 理赔 / 反欺诈模块单价上行，并在头部三家产险开展按理赔结果计费的试点。
- **控制平面契合度**：很高——理赔、核保、反欺诈正是卡片 04 点名的保险控制平面科目。
- **利润率**：毛利率稳定，经营利润率受人力扩张影响，但 AI 模块对毛利率为正贡献。
- **得分：C1✓ C2✓ C3◐ C4✓ C5◐ ≈ 4/5**。Tier A。属于"高质量项目收入"，尚未完全转为 ARR，但确有转化。

### 3.4 长亮科技（300348 SZ）
- **技术栈**：银行核心平台 + 风险 / 反洗钱模块；与华为盘古和国产大模型合作打造"AI 增强核心"；agent 层覆盖授信运营和监管报送。
- **收入信号**：2024 年头部 6 家城商行和若干股份行合同入账加速；2025 年 1H 订单中含命名的 AI 加售模块；海外（东南亚）银行核心中标也有披露。
- **计费坐标**：项目 + 模块 + 托管运营；按消费 / agent 计量在监管报送子模块中开始出现。
- **控制平面契合度**：高——银行核心、风险、监管报送。
- **利润率**：2022 年后逐步消化低毛利城商行定制收入，毛利率回升。
- **得分：C1✓ C2◐ C3◐ C4✓ C5✓ ≈ 4/5**。Tier A（下沿）。转化真实，但 AI 收入条线尚未在利润表中占主导。

## 4. Tier B — 早期变现，尚未结构化

### 4.1 宇信科技（300674 SZ）
- 银行 IT 客户根基扎实；AI agent 叙事覆盖授信、风险、渠道；与百度（文心）有战略合作 / 持股。2024 年营收随银行 IT 支出复苏增长，AI 单项收入未单独披露规模化数据，毛利率仍受二三线银行定制项目压制。C1✓ C2◐ C3✗ C4✓ C5◐ ≈ 2.5/5。

### 4.2 顶点软件（603383 SH）
- 券商 / 资管中后台业务；A5/A6 平台广泛部署于头部卖方机构；AI agent 集中在交易风控与运营；披露不充分，AI 收入未单列，客户集中度风险显著。C1◐ C2◐ C3✗ C4✓ C5✓ ≈ 2.5/5。

### 4.3 天阳科技（300872 SZ）
- 信用卡及零售银行 IT 业务；AI 风险 / 授信 agent 在头部银行试点；收入仍以传统项目为主。卡片 04 已点名零售信贷投放预算受限，封顶近期上行空间。C1✓ C2✗ C3✗ C4◐ C5◐ ≈ 2/5。

### 4.4 财富趋势（688318 SH）
- 券商交易软件厂商，零售券商粘性强；AI copilot 嵌入到向券商出货的同花顺类终端中。定价仍以单券商席位许可为主，AI 尚未独立 SKU。C1◐ C2✗ C3✗ C4◐ C5✓ ≈ 1.5/5。

### 4.5 赢时胜（300377 SZ）
- 资管和托管中后台；基金会计 / 对账 AI agent 在试点；收入受费率改革压制（卡片 04）。C1◐ C2✗ C3✗ C4✓ C5✗ ≈ 1.5/5。

## 5. Tier C — 仅有试点 / 公关披露

这些公司发布过 AI 新闻稿、加入大模型合作或在原有模块上贴 "AI+" 标签，但没有分部 AI 收入、没有计费坐标变化，毛利率改善也无法归因于 AI：

- **东方财富（300059 SZ）** — 内部大模型、零售券商 AI 助手；收入仍以经纪佣金和基金分销为主，两者均受费率压制（卡片 04）。属于内部生产力提升，不是可销售产品。C1◐ C2✗ C3✗ C4✗ C5◐。
- **大智慧（601519 SH）** — AI demo，未披露变现。
- **神州信息（000555 SZ）** — 银行 IT / 系统集成业务结构杂；AI 是营销标签；项目利润率稀释任何 AI take rate。
- **高伟达（300465 SZ）、银之杰（300085 SZ）、信雅达（600571 SH）** — 中小银行 IT 厂商，披露有限、项目占比高、无清晰的 AI SKU。
- **安硕信息（300380 SZ）** — 信用风险管理业务理论上契合控制平面，但历史执行落后于披露；需先证明 AI 模块入账再谈估值修复。
- **新致软件（688590 SH）** — 保险 IT；AI 仍是营销包装，未披露 AI 收入科目。
- **金证 / 金山云金融云板块** — 属于金融云基础设施，非金融 IT 应用厂商；归入卡片 01 基础设施层，本卡不计入。

## 6. 跨厂商的转化机制

| 转化机制 | Tier A 案例 | 销售形态 | 为什么定价能锁住 |
|---|---|---|---|
| 控制平面核心 + 模块加售 | 恒生 O45 + AI 风险模块 | 按模块许可+维护 | 客户拔不掉核心，AI 是被买入而非被砍价 |
| 私有 agent 按调用计量 API | 恒生 LightGPT、同花顺 iFinD agent | 按调用 / 按查询 | 合规约束的私有数据边界 |
| 按理赔 / 按决策结果计费 | 中科软 AI 理赔 | 按理赔可变费 | 险企可审计、可衡量的成本节省 |
| 托管型监管报送 agent | 长亮监管报送子模块 | 订阅+消费 | 监管强制、循环刚需 |
| 含 agent 能力的零售订阅 | 同花顺 Plus、财富趋势零售档 | 提价订阅 | C 端对结果有直接付费意愿 |

与卡片 01–04 串联：**真正能在 2026 年复利 AI-agent 收入的中国金融 IT 厂商，是已经拥有受监管工作流面、可在其上挂调用 / 模块 / 结果计费的那批。** 单纯的分析工具或席位许可厂商，AI 只能放大 PPT 而压缩单位经济，与卡片 02 描述的美股中腰部 SaaS NRR 压缩同源。

## 7. 反证与观察点

- **恒生 2025 年营收仍同比下滑。** 多头逻辑寄望于现金流、合同负债、毛利率领先营收 12–18 个月。如果 2026Q1 订单不见拐点，综合判断会被削弱。
- **同花顺零售依赖。** 若监管对零售 AI 投顾从目前的"许可式"转向更明文限制（CSRC / MII），C 端 Plus 收入会被直接打击。
- **中科软按结果计费仍处试点。** 险企在初期节省兑现后常缩减费用空间，按理赔合同可能在 18 个月内被压缩。
- **披露质量。** 多数 A 股金融 IT 厂商不正式分拆 AI 收入；本卡基于渠道调研、合同负债边际变化、管理层口径，噪声大于卡片 02 中美股 RPO 披露。
- **宏观尾部。** 第二轮费率改革（卡片 04）——如券商佣金封顶或公募管理费再下行——会全面压缩 IT 预算，仅留下监管强制的控制平面。

## 8. 结论

- **Tier A（恒生、同花顺、中科软、长亮）** — 这四家是 2026 年中国金融 IT 真正可信的 AI 收入标的。恒生、同花顺在披露与计费坐标上领先；中科软、长亮在控制平面契合度上领先但 ARR 化进度落后。
- **Tier B（宇信、顶点、天阳、财富趋势、赢时胜）** — 跟踪 2026Q1 订单，如 AI 收入条线突破营收 ~10% 且伴随利润率改善，再考虑重估。
- **Tier C（东方财富、神州信息、高伟达、银之杰、信雅达、安硕、新致）** — 仅有试点披露。在订单兑现前，不应支付 AI 估值溢价。

本综合呼应卡片 01–04：中国金融服务业的 agentic AI 不会蒸发 SaaS 订阅，而是把定价权集中到 Tier-A 四家厂商内部，把长尾重新打回项目集成估值。
