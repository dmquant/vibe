## Question

Investigate the topic "2027 年 HBM4 推理链条的首要弹性在合格 HBM4，首要出货瓶颈在 HBM4 验证与 CoWoS/interposer/封装基板，而液冷更多是部署瓶颈。" and make the strongest evidence-based judgment you can.

## Summary

本卡支持前序推理需求上行逻辑：288 GB HBM4 GPU 普及后，最大量价弹性在合格 HBM4 供应商，尤其是具备份额提升弹性的 MU 和具备配额权的 000660.KS；但最可能成为 2027 物理出货瓶颈的是 HBM4 验证与 CoWoS/interposer/封装基板的组合。测试设备是高经营杠杆衍生环节，液冷和机柜集成是更偏后周期的数据中心部署瓶颈。

## Report

# 2027 HBM4 推理需求：供应链与股票映射

## 立场

我支持 Card 1 的结论：reasoning-heavy inference 会把 2027 年高端加速器的 HBM-to-GPU ratio 推向约 260-300 GB。供应链含义更具体：当 288 GB HBM4 GPU 成为高端推理主流配置后，瓶颈不再只是 GPU 晶圆投片，而是合格 HBM4 堆栈、CoWoS 级先进封装和 interposer、封装基板、高速存储测试，以及可交付液冷机柜的同步供给。

## 工作模型

基准配置是 8 个 HBM4 stack 的 GPU：

| 项目 | 假设 | 含义 |
|---|---:|---|
| 单个 HBM4 stack 容量 | 36 GB，12-high | 8 个 stack 对应单 GPU 288 GB |
| 单个 HBM4 stack 带宽 | Micron 称高于 2.8 TB/s；Samsung 称最高 3.3 TB/s | 按 Micron 口径，8 个 stack 意味着单 GPU 本地 HBM 带宽高于 22.4 TB/s |
| Vera Rubin NVL72 机柜 | 72 个 Rubin GPU 和 36 个 Vera CPU | 若每个 GPU 配 288 GB，一个机柜约有 20.736 TB HBM4 |
| 每 10,000 个 NVL72 机柜 | 720,000 个 GPU | 需要 5.76 million 个 HBM4 stack 和 207.36 PB HBM4 容量 |

相较 Blackwell 时代的 HBM3E，变化不只是单 GPU 容量上升。HBM4 把 I/O interface 提高到 2,048 pins，并显著提高每 stack 带宽；同时 12-high 到 16-high 堆叠会放大散热、翘曲、键合和测试负担。SK hynix 称 HBM4 通过 2,048 个 I/O terminals 将带宽较前代翻倍，能效提升超过 40%；Samsung 称其 HBM4 稳定速度 11.7 Gbps、最高 13 Gbps，12-layer 容量 24-36 GB，并将推出 48 GB 16-layer 选项；Micron 称其 36 GB 12-high HBM4 单 stack 带宽高于 2.8 TB/s，能效较其 HBM3E 对应产品提升超过 20%。[2][3][4]

## 谁的量价弹性最大

**1. HBM 供应商的量价综合弹性最大。**  
每 1 million 个 288 GB GPU 约需要 8 million 个 HBM4 stack 和 288 PB 高端 DRAM 容量。TrendForce 预计 2026 年 HBM 总出货超过 30 billion gigabits，HBM4 在 2026 年下半年成为主流，SK hynix 仍将保持超过 50% 份额。这意味着 2027 年起点偏紧，而不是宽松。[5]

从股票映射看，最干净的上市弹性大概率是 MU，因为 HBM 相对其历史利润基数是更大的增量结构变化，而且份额提升比绝对份额更重要。000660.KS 仍有最强的早期 HBM4 执行证据和配额能力。005930.KS 的追赶期权最大：Samsung HBM4 已经商业化，采用 4 nm logic base die，并预计 2026 年 HBM 销售额较 2025 年增长超过 3 倍。[4]

**2. CoWoS/interposer 产能的瓶颈概率最高。**  
HBM4 不能单独成为可用的加速器内存，必须让逻辑 die、HBM stack、interposer 或 bridge/RDL structure、封装基板通过合格组装路径。TSMC 在年报中明确把 CoWoS、InFO、SoIC 和光子集成列为支持大规模互连以及 AI/HPC 需求的技术。[6] 在 Rubin/HBM4 周期中，如果 CoWoS-L、silicon interposer、bridge、基板或 known-good-die 处理出现短缺，已经生产出来的 HBM 和 GPU wafer 也可能无法转化为出货。

股票含义与 HBM 不同。TSMC 的稀缺价值很高，但收入基数很宽，因此盈利 beta 更稳而不是爆发式。ASE 和 Amkor 能受益于外溢订单和外协先进封装，但它们并不以 TSMC 同等程度控制 NVIDIA 级 CoWoS 核心瓶颈。UMC 应被视为 interposer/支持环节期权，而不是纯粹 HBM4 瓶颈标的。

**3. 测试设备是最高增量利润率的衍生环节。**  
HBM4 提高测试强度，因为供应商必须验证 DRAM core 质量、logic base die、TSV 和堆叠完整性、速度、热行为和最终封装质量。Teradyne 的 Magnum 7H 面向 HBM base-die wafer test、pre-singulated HBM test 和 post-singulated HBM test，并支持下一代 HBM4/4E。[7] Advantest 的 M5241 memory handler 面向 AI 时代高性能存储，计划 2026 年 Q2 开始出货，最多支持 512 个 parallel test sites。[8]

这个环节不太可能成为最大的绝对收入池，但当 HBM 层数、测试时间和验证要求同时上升时，盈利预测上修可以很明显。Advantest 的 memory-test 覆盖更广；Teradyne 的 HBM 平台叙事更清晰；FormFactor 和 MPI 对 probe interface 更敏感。

**4. 液冷和机柜集成很大，但更偏后周期瓶颈。**  
NVIDIA 称 Vera Rubin NVL72 集成 72 个 Rubin GPU 和 36 个 Vera CPU，面向大规模 reasoning、agentic AI 和 MoE inference；更早的 GB200/GB300 机柜路线已经需要高密度液冷基础设施。[1] Vertiv 的 GB200 NVL72 reference architecture 支持最高 132 kW per rack，GB300 NVL72 设计支持最高 142 kW per rack，并将在 2026 年开始提供面向 1 MW IT racks 及更高密度的 800 VDC 基础设施方案。[9]

这里的最佳股票表达取决于问题是利润率还是产能。Vertiv 和 Delta 更受益于供电/冷却标准化；Dell、HPE、Lenovo、Super Micro、Quanta、Wistron、Wiwynn 和 Foxconn Industrial Internet 更受益于机柜集成与配额。瓶颈确实存在，但通常影响 site deployment 和收入确认，早于影响 wafer-level GPU output。例外情形是 hyperscaler 已锁定半导体供应，却无法验证足够的 100 kW 以上液冷机柜、CDU、manifold、quick disconnect、leak detection 和服务团队。

## 2027 瓶颈顺序

**基准判断：** 2027 年高端推理出货先受合格 HBM4 供应和 CoWoS 级封装 slot 限制，随后受封装基板和测试产能限制，最后才是液冷数据中心部署。

1. **2027 年初：合格 HBM4 stack 供应。** 风险不是普通 DRAM 晶圆产出，而是 12-high 或 16-high HBM4 的高良率量产，包括 logic base die、TSV 堆叠、热行为以及匹配 NVIDIA、AMD 和 ASIC 平台节奏的客户验证。
2. **2027 年中：CoWoS-L/interposer/封装基板分配。** 即使内存供应改善，更大的封装和每个加速器更多的 HBM stack 会消耗稀缺先进封装面积和复杂基板产能。这是最可能的物理出货瓶颈。
3. **2027 年末：机柜冷却和供电部署。** 如果半导体供给赶上，限制因素会转向高密度数据中心 readiness：液冷回路、CDU、电源 shelf、800 VDC 迁移规划、站点 commissioning 和服务人力。

## 结论

最大量价弹性在合格 HBM4 供应商，尤其是具备份额提升弹性的 MU 和具备存量配额权的 000660.KS。最可能成为 2027 出货瓶颈的，仍是 CoWoS 级先进封装、interposer/封装基板可得性与 HBM4 验证的组合，而不是单独的液冷。只有当半导体封装已经足量可用、hyperscaler 开始把 GPU 配额转化为通电机柜时，液冷才会成为更显性的瓶颈。
