## Report

# 卡片 03 - A股与港股 TMT 公司中，谁能真正捕获 DeepSeek V4 下游 Agent 与微调经济性

**分析师:** TMT 行业分析师
**立场:** synthesize（综合）
**日期:** 2026-04-30
**卡片:** 3/8 - 根主题: DeepSeek V4 国产化生态链

## 1. 核心判断

我综合并延伸卡片 02 给出的权重（模型微调 + Agent 应用层 45%）。落到上市公司层面，A股与港股 TMT 中真正能把 V4 低成本长上下文推理转化为可持续利润的标的并不多。能做到的公司必须同时具备三个特征：

1. **专有任务数据**：模型厂商无法自由抓取的工作流日志、行业结构化模板、客户交易数据、边缘设备遥测等。
2. **嵌入付费工作流的分发**：按席位或按流程可衡量 ROI，而不只是 MAU。
3. **可计量的 recurring agent runs**：通过订阅、按席位 AI 加购、行业 API 按调用计费等方式产生持续收入，而不是一次性集成项目收入。

按这三个过滤器筛 A 股 + 港股 TMT，得到的 Tier A 名单很短（3-4 家），Tier B 仍需用季报兑现，Tier C 多为概念暴露，Tier D 应回避。

**Tier A（最有可能把 V4 经济性转化为利润）：** 金山办公（688111.SH）、广联达（002410.SZ）、美图公司（1357.HK）。
**Tier B（站位不错，变现仍待证明）：** 金蝶国际（0268.HK）、用友网络（600588.SH）、恒生电子（600570.SH）、深信服（300454.SZ）、中科创达（300496.SZ）。
**Tier C（概念敞口、单席位经济性弱）：** 科大讯飞（002230.SZ）、三六零（601360.SH）、神州数码（000034.SZ）、浪潮信息（000977.SZ），以及多数纯系统集成商。
**Tier D（仅有重估、没有盈利锚）：** 单纯模型套壳、GPU 出租壳、纯概念的 AI 应用题材。

核心洞察：V4 降低了 **每次 agent run 的成本**，但本身并不创造 **变现界面**。变现界面属于已经掌握用户、文档、合同、施工模型、医疗记录、产线的人。这意味着这条线本质上是 **既有垂直 SaaS 公司穿上 V4 成本经济性的外衣**，而不是新一批"AI 应用"公司从零起来的故事。

## 2. 为什么过滤器必须这么严

卡片 01 已确立 V4 × DCU-3 是 Tier-2 软件栈协同优化，而非架构级联合设计。卡片 02 则得出最大未定价弹性在应用层（45% 权重），原因在于更便宜的长上下文、可工具调用的推理会扩张企业工作流中的 token 用量。

但有两个结构性事实，使得上市公司中绝大部分"AI 应用"无法捕获这部分弹性：

1. **开源权重 + 极低 API 价格压缩了套壳层利润。** DeepSeek 官方价目表为 V4-Flash 输出 $0.28 / 1M tokens，V4-Pro 75% 促销价 $0.87，V4-Pro 标准 $3.48，已经把任何转售方的边际成本下限钉死。没有专有数据、没有分发、没有工作流锁定的纯套壳公司无法持有可持续价差。
2. **真正能变现的是 agent run-rate，而不是概念数。** 付费单元是"完成的工作流 / 月 / 席位"或"成功 tool-call / 月 / 客户"。没有可量化 ROI 的工作流，就没有定价的位置。

这就是 Tier A 名单为何这么短，也是为什么 A 股最猛的 AI 主题行情（系统集成商、服务器经销商、GPU 云壳）应当被降权——它们承接的是流量，不是工作流利润。

## 3. Tier A —— 现实层面能把 V4 转化为利润的名字

### 3.1 金山办公（688111.SH）—— WPS AI 的席位经济性

金山办公 2025 年报披露 WPS 个人订阅累计付费用户 4170 万，月活 6.32 亿，AI 能力已贯穿 WPS 全套件。WPS AI 以个人按席位加购、企业 SKU 双轨变现。

V4 对利润的传导：
- WPS 内部的文档语料、模板与编辑轨迹，是中国最接近微软 365 Graph 的私有数据资产，正是模型厂商无法自由抓取的专有任务数据。
- V4 更便宜的长上下文推理直接拉低 WPS AI 每次调用的 COGS（数百页合同审阅、依据长报告生成幻灯片、对真实工作簿的电子表格 Agent）。
- 分发已经在收钱：WPS 个人订阅本身已变现，AI 是 ARPU 增量，而不是免费拉新。

风险：WPS AI 定价权取决于用户是否愿意为 AI 单独付费，以及 WPS AI 与免费豆包 / 元宝消费端的差距能否拉开。投资逻辑是 ARPU 抬升，而不是看 DAU 标题。

### 3.2 广联达（002410.SZ）—— 建筑 BIM 工作流即 Agent 基底

广联达是国内建筑软件龙头，沉淀了覆盖造价、BIM、项目管理的工作流数据，2024-2025 披露已在造价与项管产品中嵌入 AI 能力。

V4 对利润的传导：
- 建筑工作流天然是长上下文（数百张图纸、规范、清单项、历史项目）+ 工具调用（造价库、进度引擎、BIM 浏览）。V4-Pro 1M 上下文与降低后的 KV-cache 形状刚好匹配。
- 数据壁垒（省级造价库、施工方专属模板）确实难以从公开数据复制。
- recurring agent runs 可直接挂在已有的按席位 / 按项目 SaaS 收费上。

风险：建筑业资本开支偏弱；命题在于席位基数平甚至下行的前提下能否抬 AI ARPU。Tier A 是基于能力，而非宏观顺风。

### 3.3 美图公司（1357.HK）—— 已规模化的 AI 订阅 P&L

美图 2025 年中报披露付费订阅用户 1440 万，同比 +39.8%，图像 / 视频生产力 SKU（美图设计室、Wink、美图秀秀）已直接对 AI 生成功能收费。这是港股中国消费互联网中最干净的"AI 订阅 P&L"。

V4 对利润的传导：
- 美图采用多模型编排，长上下文推理单 token 成本下降直接改善"理解 prompt + 检索素材 + 编辑"创意 Agent 流程的毛利。
- 私有用户图像数据与编辑历史，给到垂直创意任务的微调优势。
- 是当前港股 AI 订阅 P&L 最纯粹的标的之一。

风险：豆包图像、腾讯混元创意、阿里通义万相的竞争压力；用户留存对免费档延伸的脆弱性。

## 4. Tier B —— 站位不错，但变现仍在观察期

| 代码 | 名称 | 为何是 Tier B 而非 Tier A | 未来 2-4 季度需要看到的证明 |
|---|---|---|---|
| 0268.HK | 金蝶国际 | ERP 工作流数据 + 苍穹平台；AI 加购已披露但单席位 ARPU 抬升尚未单独披露 | AI 单席位 ARPU；苍穹 Agent 客户数 |
| 600588.SH | 用友网络 | A 股最大 ERP 装机量；YonGPT 与 BIP Agent 平台；历史转化偏慢 | YonGPT 付费客户数；毛利率走向 |
| 600570.SH | 恒生电子 | 持有金融机构后台工作流（基金行政、券商）；专有交易流程数据 | 面向资管的 Agent SKU 定价；持续性收入占比 |
| 300454.SZ | 深信服 | 安全运营 Agent、云托管服务；专有攻击遥测数据 | MDR / Agent 增强 SOC 的 ARR 增速 |
| 300496.SZ | 中科创达 | 边缘 / 汽车 Agent 运行时；专有 IoT 与座舱遥测 | 主机厂客户单车或单设备 AI 收入披露 |

这些公司至少具备 Tier A 三大特征中的一项，但缺少可见的单席位 AI 收入，或者历史转化偏慢。打分方式不应看 demo 数，而应看 **下两个季报里是否单列 AI 收入科目，且边际毛利为正**。

## 5. Tier C —— 概念敞口，单席位经济性弱

- **科大讯飞（002230.SZ）**：拥有教育与语音专有数据、真实分发，星火 X1 上的 Agent 叙事也在跑。但讯飞同时投资自有基础模型，与 V4 经济性是替代而非搭载关系，损益表承担训练成本。属于板块敞口，不是干净的 V4 变现者。
- **三六零（601360.SH）**：品牌可见度高、有消费 Agent 产品，但单用户 AI ARPU 偏小，更多是概念驱动。
- **神州数码（000034.SZ）、浪潮信息（000977.SZ）等多数纯系统集成商**：能在 DCU-3 与 DeepSeek 部署中拿项目收入，但项目毛利薄、波动大。这一段计入卡片 02 的 OEM / SI 18% 权重，而不是应用层 45%。

Tier C 是指数层面"DeepSeek 概念"成交量的主力。可以作为主题 beta 持有，但 **不是 V4 应用层利润命题的主仓位**。

## 6. Tier D —— 回避

- 没有专有数据、没有工作流嵌入的纯 DeepSeek API 转售壳。
- AI 收入披露 < 5%、AI 毛利不披露的概念股。
- 没有合同利用率的 GPU 出租壳。

这些名字会随标题重估，但下一次 token 价格下调一来，就没有保留毛利的机制。

## 7. 对卡片 02 应用层 45% 权重的具体表达

45% 在主题层面是合理的，但落到上市公司层面集中度极高。可防守的应用层组合表达大致是：

- **核心仓位（应用层 sleeve 的 60-70%）：** 金山办公、广联达、美图——当下就有专有数据 + 付费工作流界面的名字。
- **观察仓位（20-30%）：** 金蝶、用友、恒生电子、深信服、中科创达——只有当下一份季报单列 AI 收入科目时再加仓。
- **题材敞口（0-15%）：** Tier C 作为主题 beta，仓位小、按概念而非盈利做风险预算。
- **回避：** Tier D 全数。

这与卡片 02 的视角形成杠铃：制造端配 HBM3 / 先进封装，应用端配少而专、专有数据厚的几个名字，避免把系统集成 beta 当成应用层 alpha 来买。

## 8. 观察点

1. **2026 H1 报告中金山办公、金蝶、用友的单席位 AI ARPU 披露**——判断 V4 时代的推理经济性能否兑现为应用层利润的最关键单一指标。
2. **美图付费订阅用户与 ARPU 走向（2026 中报）**——验证订阅式 AI 是否仍以 >30% 同比复利。
3. **广联达造价 / BIM 产品的 AI 功能附加率**——作为建筑业资本开支的函数。
4. **DeepSeek 自身的企业 / Agent 产品路线图**——若 DeepSeek 自己做法律 Agent、金融 Agent，会蚕食 Tier A、Tier B 厂商的同一界面。
5. **大厂捆绑**——若阿里 / 腾讯 / 百度把 Agent 运行时随云免费捆绑，垂直 SaaS 定价权将被压缩。

## 9. 交接

下一个未解问题已从"哪些公司符合命题"切换到"在当前估值与拥挤度下，这一观点应如何在组合中表达"。这是板块配置与风格轮动问题，而非更深层的 TMT 基本面问题。建议交接给 `chief-strategist` [primary]，把 Tier A / B / C / D 名单转化为 A 股 + 港股配置内的仓位化篮子，并标注哪些已显示拥挤、哪些应用层 alpha 仍未被定价。
