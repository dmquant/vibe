# QA治理审查：TMT路由权重的所有权、变更流程、语料刷新与审计证据

## 结论

**立场：综合。最终QA评级：生产部署前需要修订。**

我支持第1至第7张卡片的研究结论：过宽的AI/TMT语言正在把`tmt-analyst`变成兜底入口，而第7张卡片验证的权重向量与**Delta = 0.25**双负责人阈值，方向上已经具备受控部署价值。数字链条内部一致：第2至第3张卡片识别出**80**条样本中有**56**条误入TMT，TMT误分率为**70.0%**，其中**12**条US Tech/Mag 7内容更适合`offshore-strategist`；第7张卡片报告，在100条压力语料上，TMT误分率降至**15.4%**，改善**54.6个百分点**，双负责人触发率为**11%**，低于**15%**过度触发警戒线。[C1][C2][C6]

QA评级不是失败，因为研究推理主线连贯，拟议控制也可落地。但评级为**需要修订**，原因是实时工作区缺少多个被引用的证据文件，包括`session-brief.md`、`analyst-catalog.json`、`card-01`、`card-05`和`card-07`；因此本次审查中第7张卡片只能依据任务提示中提供的会话快照。生产部署不能依赖记忆或聊天上下文。证据包必须恢复并版本化，之后该路由权重才能被视为可审计的运营控制。[C8]

## 必需的所有权模型

权重不应由任何单一分析师邮箱独占。路由向量是会改变工作量、研究覆盖和升级行为的共享运营控制。所有权应拆分如下：

| 职能 | 负责人 | 责任 |
| :--- | :--- | :--- |
| 策略所有者 | `chief-quant` | 负责评分方法、权重向量、Delta阈值、敏感性测试和模型变更工单。 |
| 业务所有者 | 研究运营负责人或路由产品负责人 | 负责生产部署、回滚权限、监控看板和事件分诊。 |
| 领域审批人 | `tmt-analyst`、`offshore-strategist`、`global-macro`、`chief-strategist` | 审批影响各自覆盖范围的领域标签、排除规则、硬规则和示例。 |
| 合规审查人 | `compliance-officer` | 审查限制清单、制裁、披露和客户可执行性触发器；不成为默认研究负责人。 |
| QA闸口 | QA Manager | 核查文件完整性、数字一致性、双语一致性、证据文件和发布签批。 |

核心治理规则是：**`tmt-analyst`是受影响方，不是唯一所有者。** 该变更会把流量重新分配给`offshore-strategist`、`global-macro`和`chief-strategist`，因此这些负责人必须审批自身标签定义和失败容忍度。

## 变更流程

每一次路由变更都应通过带版本号的变更工单处理，并分为四类严重级别。

| 变更类别 | 示例 | 最低审批 |
| :--- | :--- | :--- |
| Class 0：仅文档 | 澄清示例、错别字修正、引用更新 | QA Manager |
| Class 1：低风险词表更新 | 新增同义词，且保留集指标变化不超过**1个百分点** | `chief-quant`加受影响领域负责人 |
| Class 2：评分或阈值变更 | 权重更新、将Delta从**0.25**改为其他值、新增双负责人条件 | `chief-quant`、业务所有者、QA Manager和所有受影响领域负责人 |
| Class 3：硬规则或分类体系变更 | 芯片加出口管制强制双负责人、新增负责人类别、合规升级规则 | Class 2审批人加`compliance-officer` |

发布流程应为：

1. 打开变更工单，写明负责人、原因、范围、预期路由影响、回滚方案和受影响分析师。
2. 冻结评估语料版本，并附上精确的候选`weight-vector.json`。
3. 用当前基线和候选向量运行离线回放。
4. 对所有变化样本或争议样本进行独立标签复核。
5. 部署前检查验收闸口。
6. 分阶段发布，先进入影子模式，再进入有限生产，最后全量生产。
7. 归档发布包，包含校验和、审批、监控结果和回滚状态。

## 语料刷新节奏

证据语料应被视为持续更新的审计资产，而不是一次性回测。

| 节奏 | 必需动作 |
| :--- | :--- |
| 每个路由项 | 记录前三名分析师分数、胜出负责人、Delta差距、双负责人标记、硬规则触发器、时间戳和匿名化提示哈希。 |
| 每周 | 复核至少**30**条近期路由、全部双负责人覆盖、全部人工改派，以及所有分数差距不高于**0.25**的项目。 |
| 每月 | 新增**40-60**条已标注生产样本，并在TMT基本面、Mag 7/ADR、美国AI政策、A股市场结构、AI主题和相邻行业之间保持平衡。 |
| 事件驱动 | 重大出口管制、对外投资、A股融资融券、解禁、上市规则或Analyst Catalog变化后，在**5个工作日**内刷新触发器。 |
| 每季度 | 对分层审计样本重新标注，并围绕**0.20**、**0.25**和**0.30**重新运行Delta敏感性分析。 |
| 每半年 | 完整治理审计，覆盖所有权、审批、证据留存、覆盖日志和漂移指标。 |

这一节奏针对的是本线程的主要漂移风险：AI、芯片、云、平台、出口管制和市场结构词汇变化很快，过期样本会逐步重新制造TMT语义引力井。

## 证据文件标准

每次变更都必须包含一个发布文件夹，并包含以下文件：

| 文件 | 必需内容 |
| :--- | :--- |
| `change-ticket.md` | 变更原因、申请人、受影响负责人、类别、风险评估、回滚方案和部署窗口。 |
| `weight-vector.json` | 分析师权重、Delta阈值、硬规则、特征词典、版本、作者、时间戳和校验和。 |
| `eval-corpus.jsonl` | 项目ID、匿名化交接文本或哈希、来源期间、黄金负责人、允许的第二负责人、标签理由、标注人、复核人和PII标记。 |
| `offline-results.json` | 混淆矩阵、各负责人precision/recall、TMT误分率、双负责人率、假阳性、假阴性和子集指标。 |
| `sensitivity-report.md` | 对**0.20**、**0.25**和**0.30**的Delta扫描，以及为什么**0.25**仍是运营阈值的解释。 |
| `domain-signoff.md` | `tmt-analyst`、`offshore-strategist`、`global-macro`和`chief-strategist`签批；Class 3变更需要`compliance-officer`签批。 |
| `qa-review.md` | 文件完整性检查、数字一致性检查、适用时的双语一致性检查、引用检查和最终QA评级。 |
| `monitoring-plan.md` | 影子模式周期、生产发布步骤、告警阈值、回滚触发器和当班负责人。 |

当前线程尚未达到该证据标准，因为第7张卡片不在实时文件系统中，第5张卡片在会话简报中也呈现为损坏的交接摘要，而不是可用的合规报告。这些缺陷必须在审计签批前修复。[C8]
