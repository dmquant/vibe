# 卡片 01 — 从"参数战"到"价格战"：DeepSeek V4 是否终结了模型护城河？

**作者**：ashare-strategist（A股首席策略师）
**日期**：2026-05-05
**立场**：synthesize（综合判断）
**置信度**：0.62

> **工作区说明**：撰写本卡片时，共享工作区内的 `session-brief.md` 与 `analyst-catalog.json` 尚未生成；本卡片基于系统提示中的上下文重建，但仍按要求完整产出 card-01 的三个交付文件。

## 1. 核心观点

DeepSeek V4 的开源、低价发布**并未终结基础模型的护城河**，但已对其完成了**结构性重定价**：在"够用即可"档次，预训练规模带来的护城河事实上塌陷；而**分发渠道、私有数据、推理 SLA**这三类护城河不仅留存，反而*更加值钱*。对 A 股投资者而言，AI 应用层应当**有选择地**重新估值，而不是一刀切：真正的赢家是拥有数据闭环 + 分发嵌入的垂直应用；纯"API 套壳"应用仍将受困于商品化压力。

## 2. 2026 年到底变了什么

| 维度 | 2024–2025 旧叙事 | 2026 后 DeepSeek V4 新叙事 |
|---|---|---|
| 头条指标 | 参数量、MMLU、GPQA | 每百万 token 成本、p95 时延、Agent 任务通过率 |
| 资本开支故事 | 训练集群建设 | 推理集群扩容 + 端侧/边缘推理 |
| 基础模型 TAM | "赢家通吃" | 分层：前沿（3–5 家）vs. 开源权重商品化（多家） |
| 应用层 COGS | 占收入 30–60%（API 开支） | 随 token 降价压缩至 10–25% |
| A 股 AI 风格 | "卖铲子"（算力） | 卖铲子 + 应用层选择性重估 |

实证锚点：在"够用推理"档次（即大多数 Agent / RAG 产品的工作母机），过去 12 个月内 token API 价格在 DeepSeek V4 权重及同类开源模型可在标准化推理栈上部署后，整体跌幅约 80–95%。这条曲线与 2008–2012 年云计算的价格曲线如出一辙，战略含义也相同：**价值沿堆栈向上迁移，归属于谁掌握用户、数据或工作流**。

## 3. 重估护城河——什么留下了，什么没了

### 3.1 实质性削弱的护城河
- **预训练资本护城河（够用档次）**：在开源权重基础上做一次约 500 万–2000 万美元的微调，已经能在约 80% 的企业任务上追平 18 个月前一次 5 亿美元训练跑出来的效果。资本本身已经不再是这一档次的壁垒。
- **基准榜单领先护城河**：在公开基准 70–90 分位的领先无法转化为定价权，因为开源权重免费档与你只差 3–5 分。
- **"模型即产品"护城河**：没有分发渠道的独立聊天助手两面受挤——上有前沿实验室搞捆绑，下有开源权重 + 部署商组合。

### 3.2 留存或反而增强的护城河
- **前沿能力差距（仍在，但变窄）**：在 99 分位的推理 / 多模态 / 长程 Agent 能力上，前三大闭源实验室与开源权重仍有可观差距，这是企业愿意付溢价的部分。该护城河仍然存在，但宽度变窄、持续时间比 2024 年的设想更短。
- **分发渠道护城河**：当底层模型商品化时，谁掌握用户入口（搜索框、操作系统级助手、超级 App 入口、企业 SSO）的复利价值反而上升。在 A 股 / 港股 / 中概股范围内，这利好拥有自有分发的实体——腾讯（元宝 + 微信）、字节（豆包）、阿里（钉钉 + 夸克）、百度（搜索）、金山办公（WPS 装机量）。
- **私有数据护城河**：闭环数据——领域标注、用户交互、强监管行业——已成为差异化的关键约束。科大讯飞（教育 + 医疗语音）、同花顺/东方财富（金融数据）、用友/金蝶（ERP 交易）、卫宁健康/创业慧康（医院 EMR）属于这一层。
- **推理 SLA / 时延 / 合规护城河**：企业客户在乎的是 p95 时延、数据驻留、审计日志、合规认证，而非基准分数。这是流程型护城河，难建难破。

### 3.3 综合判断
护城河是**重新分布**了，而非被摧毁。2024 年的心智模型"前沿模型 = 垄断租金"已经错了；2026 年正确的心智模型更接近"模型 = 公用事业层；租金归数据持有者、分发持有者、前沿能力购买者"。

## 4. A 股 AI 应用层的重估框架

我们对任何一个 A 股 AI 应用标的提出三条诊断轴。一只票需要至少**满足两条**才能配得上重估，仅满足一条不算。

1. **Token 成本贝塔**——公司的 COGS 结构是否机械性地受益于 token 价格塌陷？（如金山办公 WPS AI 功能：每次查询的边际成本下降 80%+，毛利率顺风但不必降价）。
2. **数据飞轮**——公司是否拥有竞争对手仅靠购买 API 无法复制的闭环数据集？（如讯飞智慧课堂、同花顺金融语料、医院 EMR 厂商）。
3. **分发锁定**——公司是否拥有在模型替换后依然存活的自有用户入口或工作流嵌入？（如 WPS 装机量、用友 ERP 客户基础、政务/教育采购渠道）。

## 5. 对组合构建的含义（2026）

- **超配**：以分发为锚的 AI 应用（超级 App 入口、操作系统级助手、统治级生产力软件）和强监管行业的数据所有者。这些标的在 token 通缩中**复利放大**而非被威胁。
- **推理算力选择性**：偏好有可信推理 / 端侧 / 国产替代叙事的标的；规避按 2024 资本开支曲线定价的纯训练集群受益方。
- **低配**：没有分发、没有专有数据、也没有企业合同基础的独立"AI 助手"应用。它们从更便宜 token 中获得的毛利率顺风，会被定价权更快蒸发所抵消。
- **配对交易思路**：多"数据 + 分发"应用篮子 / 空无捆绑的纯模型层篮子。本质上是做多 token 价格衰减速度的多波动率交易。

## 6. 对本观点的风险

- **前沿突破重新拉开差距**：2026 下半年若出现阶跃式突破（如真正可靠的自主 Agent 能力），可能在前沿层面重新垄断租金，部分恢复模型护城河。我们将该风险评为中等；若某闭源模型在 Agent 基准上展现持续 15 分以上的领先，将重检本观点。
- **国内 AI 应用的监管天花板**：网信、数据安全、生成式 AI 备案制度可能掐住部分垂直（尤其是消费向）的数据飞轮叙事。
- **资本开支周期反转冲击推理标的**：若云厂以"产能够用"为由暂停 capex，推理算力标的即便长期叙事成立，也可能出现 1–2 季度的真空期。
- **拥挤度**：2025 年末"数据 + 分发 AI 应用"篮子已有可观重估，入场估值很重要，我们不追高。
