## 一、终局判断（核心论点）

**AI 生产力工具组合在 2026H2 应保持"结构性 OW + 纪律化降风险"的双轨姿态：在估值层（实际利率/美元）和基本面层（cRPO/AI ARR/毛利率/资金流）任一进入红色档前，超配平台型与工作流型 SaaS 重估篮子；一旦 Trigger 0 或 Trigger 1+3 同时触发，则按已固化的"衍生品桥接 + 分日现货压降"路径，把 China AI Cloud 从 27% 强制压至 16%，并把现金缓冲提升至 22%。**

这一结论是七张前置卡片的逻辑闭环：
- **业绩兑现（卡 01）**确认了"选择性重估"假设——只有把 AI 能力嵌入数据、工作流和交易闭环的平台才进入 ARR/cRPO 兑现期；
- **可投资篮子（卡 02）**把这一基本面判断转化为美股 + 港股的具体名单；
- **配置与触发器（卡 03）**把名单转化为权重、风险预算与四级再平衡纪律；
- **宏观压力测试（卡 04）**指出基本面触发器对估值杀跌滞后，必须前置 Trigger 0（TIPS+DXY）；
- **风控（卡 05）**指出尾部情景下对冲不足，必须强制现货降仓；
- **执行（卡 06）**给出"D0 衍生品先到位 + 6–10 日低参与率分布卖出"的路径；
- **衍生品（卡 07）**给出"锁 Delta + 盈余 Gamma + 修正基差"的三层桥接结构。

终局结论是把这七步整合成一份可执行、可审计、可动态调整的纪律清单。

## 三、四档纪律状态机（Green / Yellow / Orange / Red）

整合 Trigger 0（宏观估值）+ Trigger 1–4（基本面 / 资金流），形成单一状态机：

### 🟢 Green（基线）
- TIPS 10Y 实际收益率 ≤ 2.30%，DXY ≤ 105，MOVE ≤ 110
- cRPO 增速 ≥ 收入增速；AI ARR 口径稳定；毛利率 QoQ -50bp 以内；南向资金 20D 净流入 > 0
- **动作**：维持基线权重；对冲层保持 5% 名义；月度再平衡

### 🟡 Yellow（前置预警，Trigger 0 单独触发或 Trigger 4 单独触发）
- TIPS 10Y > 2.30% 或 DXY > 105，或南向资金 20D 净流入由正转负
- **动作**：
  1. 把 QQQ/HSTECH Put Spread 名义从 5% → 8%
  2. 现金从 15% → 18%（资金来源：算力链 -2%，工作流卫星 -1%）
  3. 不动核心 SaaS 与 China AI Cloud 现货
  4. 启动每日 TIPS、DXY、HSTECH 隐含波动率监控

### 🟠 Orange（基本面恶化，Trigger 1 或 Trigger 3 单独触发）
- 任一核心持仓 cRPO 增速 < 收入增速持续 1 个季度，或加权毛利率 QoQ ≤ -100bp
- **动作**：
  1. 对触发个股按 -25% 权重压降，资金归现金或对冲层
  2. China AI Cloud 上限从 22% 降至 19%
  3. 审视 AI ARR 披露口径变化（卡 03 优先级 #2）
  4. 投委会每周复盘

### 🔴 Red（复合尾部，Trigger 0 + Trigger 1/3 同时触发，或卡 05 红色情景）
- TIPS > 2.50% **且** DXY > 108 **且** MOVE > 130 **且** 港股流动性折价 > 8%
- 或：China AI Cloud 篮子加权毛利率 QoQ ≤ -200bp
- **动作（执行卡 05/06/07 全套）**：
  1. **D0**：用 HSTECH 期货空 + 5/10 OTM Put Spread 把 China AI Cloud 有效 Beta 从 27% → 16%–19%（卡 07 三层结构）
  2. **D1–D8**：按 9988.HK / 9888.HK 主力卖出 + 1024.HK / 3896.HK 慢速 + 0700.HK 校准的分日路径压降现货（卡 06）
  3. 现金缓冲提至 22%；对冲层名义提至 8–10%
  4. 实现短缺 > 150bp 或基差失控 → 暂停现货卖出，升级投委会

## 七、风险与剩余不确定性

1. **Trigger 0 阈值的校准误差**：TIPS 与 DXY 的阈值依赖 Fed 点阵与全球美元流动性，本卡采取卡 04 给出的中性档位；2026H2 若 Fed 再现意外鹰派，需要在 Yellow→Orange 之间增设额外档位。
2. **AI ARR 口径风险**：核心持仓中部分公司（特别是 Adobe、Salesforce）对 AI ARR 披露口径仍在演化，若大型厂商集体下调披露，可能制造一次性集体杀估值。卡 03 已把"披露退化"列为 Trigger 2，本卡保持。
3. **执行风险**：Red 档第 1 步衍生品桥接依赖 HSTECH 期货 + 个股 Long Put 的可获得性；流动性闪崩时保证金占用可能急升，需提前与 Prime Broker 锁定 25%–30% 的额外保证金额度。
4. **政策尾部**：AI 芯片出口管制、跨境数据合规、港股流动性政策都可能制造非线性冲击；本框架假设它们触发"事件驱动"档位，但若同时出现，组合可能短暂失去执行窗口。

## 八、结语：何时停止"重估"叙事

当以下任一条件出现，应当**主动退出**该 AI 软件重估篮子，而非仅仅降权重：
- 加权 cRPO 增速连续 2 季度低于收入增速 5pp 以上 → 选择性重估假设证伪
- 加权毛利率 QoQ ≤ -300bp 持续 1 季度 → 推理成本吞噬利润假设证实
- 南向 + 北向资金对篮子核心同时净流出超过 200 亿 RMB 1 个月 → 资金面共识破裂
- Fed 路径转向 + 实际利率永久性站上 3% → 长久期 SaaS 估值范式终结

在上述四条均未触发前，本框架保持"结构性 OW + 纪律化降风险"的双轨姿态。组合的 alpha 来自于"选择性识别"，组合的 beta 防御来自于"分档纪律"，二者缺一不可。
