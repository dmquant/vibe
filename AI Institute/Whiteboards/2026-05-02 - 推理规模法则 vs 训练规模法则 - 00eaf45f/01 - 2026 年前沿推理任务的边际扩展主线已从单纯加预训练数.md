## Question

Investigate the topic "2026 年前沿推理任务的边际扩展主线已从单纯加预训练数据转向 test-time compute，2027 年硬件需求将体现为 HBM/GPU 配比上行。" and make the strongest evidence-based judgment you can.

## Summary

本卡有条件支持命题：到 2026 年，对推理密集型前沿 AI 而言，增加 test-time compute 的边际能力回报已高于单纯增加人类预训练文本数据，但预训练仍决定基础上限。硬件上，2027 年新增需求将更偏向推理和 agentic serving，GPU 需求仍强，但 HBM bit、带宽和机架级内存架构会成为更紧约束。基准判断是 2027 年 HBM bit 需求增速约为加速器台数增速的 1.5-2.0x，新增加速器小时中推理占 55-65%。

## Report

# 推理计算 vs. 预训练数据：2026 年 TMT 视角

**本卡立场：** 初始综合判断，对命题作有条件支持。

## 核心结论

在 2026 年的算力环境下，对前沿推理任务而言，增加 test-time compute 的边际回报大概率已经超过了单纯增加人类预训练文本数据。这不等于预训练不重要。基础模型规模、数据质量、多模态数据、合成数据以及强化学习训练仍然决定能力上限。但在数学、代码、科学推理、agentic planning、长上下文检索和多步骤工作流中，新的生产前沿已经很清楚：先训练足够强的基础模型，再通过更长推理轨迹、并行采样、验证器引导搜索和工具调用，在推理阶段自适应地花更多算力。

硬件含义也很直接：2027 年 AI 基础设施需求会在每单位加速器算力上变得更加“重内存”。GPU 需求仍然强，因为更多推理意味着更多 token 和更多服务时长；但 HBM 需求应快于 GPU 台数增长。我的基准判断是，2027 年新增部署的加速器小时中，推理和 agentic serving 约占 55-65%，训练约占 35-45%；而新增 HBM bit 需求中，推理相关占比可能达到 60-70%，原因是长上下文和 KV cache 密集型 decode 对内存容量与带宽更敏感。这是基于下文模型证据和硬件路线图得到的内部估算，不是公司披露口径。

## 为什么边际扩展杠杆发生变化

过去的训练规模法则主要基于 Chinchilla 式 compute optimality：给定训练算力预算，模型参数和训练 token 数应大致同步提升。DeepMind 的 Chinchilla 论文明确发现，在其假设下，最优模型规模和训练 token 数都大约按 C^0.5 扩展 [S1]。这个框架对 next-token 预训练损失是正确的，但没有充分计入部署阶段：一个模型可能服务数十亿次请求，并且可以在推理阶段对不同问题投入可变算力。

到 2026 年，两个摩擦削弱了“只加原始预训练数据”的吸引力：

1. 公共人类文本数据不再是容易指数扩张的资源。Epoch AI 估算，经质量和重复度调整后的人类生成公共文本存量约为 300 万亿 token，90% 区间为 100 万亿至 1,000 万亿 token；若趋势延续，2026 至 2032 年间可能被充分使用 [S2]。如果按推理经济性优化而选择过度训练，实际数据墙还可能更早出现。
2. 推理任务不能只靠记忆更多网页文本解决。它们受益于搜索、验证、拆解、自我纠错和重复尝试，而这些机制消耗的是推理算力，不只是预训练 token。

test-time compute 方向已经有直接经验证据。OpenAI 报告称，o1 的表现会随着强化学习训练算力和测试时“思考”时间增加而提升 [S3]。随后 OpenAI 表示，o3 在训练算力和推理阶段 reasoning 上都又推进了一个数量级，同时仍看到明确收益，并且 o3/o4-mini 相比此前 o-series 模型改善了成本-性能前沿 [S4]。ICLR 2025 论文《Scaling LLM Test-Time Compute Optimally》发现，自适应 test-time compute 的效率可比 best-of-N 采样高出 4x 以上；在小模型已有非零成功概率的问题上，小模型加 test-time compute 可以胜过 14x 更大的模型 [S5]。DeepSeek-R1 从另一角度验证了同一方向：强化学习可以诱发自我反思、验证和动态策略调整，这些推理轨迹还可蒸馏到更小模型中 [S6]。

关键限定是：效果具有领域依赖性。test-time compute 最适合模型能够搜索候选推理路径、并识别或验证进展的场景。若缺少事实知识、检索质量低、任务定义差，或基础模型初始成功概率接近零，推理阶段多花算力的效果会弱很多。因此正确结论不是“推理算力在所有地方都胜过训练数据”，而是“在 2026 年的前沿推理工作负载中，只要已有足够强的基础模型，边际能力提升越来越由推理算力主导”。

## 硬件映射：从训练吞吐到推理吞吐

训练集群优化的是密集矩阵吞吐、all-reduce 网络、利用率以及 checkpoint / 数据流水线。推理服务集群会额外遇到不同瓶颈：

- 更长的内部推理轨迹会增加每次用户回答生成的 token 数。
- 并行采样和验证器引导搜索会在最终答案前生成多组候选 token。
- Agentic 工作流会加入工具调用、检索、代码执行、记忆读取和多轮状态。
- 长上下文服务会增加 KV cache 内存占用，尤其是在 decode 阶段。
- Decode 往往比 prefill 更受内存带宽约束，因此 HBM 带宽和容量与原始 tensor FLOPS 同样重要。

这一点已体现在产品表述和规格中。NVIDIA 称 DGX B200 相比 DGX H100 训练性能为 3x、推理性能为 15x；其 2026 年 4 月引用的 Blackwell 推理成本约为 GPT-OSS-120B 每百万 token $0.02 [S7]。Vera Rubin 被明确定位于长上下文和 agentic 系统，Rubin 产品将于 2026 年下半年由合作伙伴供货 [S8]。AMD MI350 也沿同一方向推进，单 GPU 配置 288GB HBM3E 和 8TB/s 带宽 [S9]。

## HBM/GPU 配比：2027 年需求含义

最清晰的数量信号来自每代加速器的 HBM 容量和带宽：

| 平台 / 参考点 | GPU 内存与带宽 | 需求含义 |
|---|---:|---|
| NVIDIA H100 SXM | 80GB HBM，3.35TB/s 带宽 [S10] | Hopper 时代训练和早期 LLM 推理基准。 |
| NVIDIA DGX B200 | 8 颗 GPU 合计 1,440GB HBM3e、64TB/s 总带宽，折算单 GPU 180GB、8TB/s [S7] | Blackwell 单 GPU 内存容量约为 H100 的两倍，并显著提升推理效率。 |
| HPE 版 NVIDIA GB200 NVL72 | 72 颗 Blackwell GPU，最高 13.5TB HBM3e、576TB/s 带宽 [S11] | 机架级共享 HBM 成为 AI factory 的基本单元。 |
| NVIDIA Vera Rubin NVL72 | 72 颗 Rubin GPU，20.7TB HBM4、1,580TB/s 带宽；单颗 Rubin GPU 为 288GB HBM4、22TB/s 带宽 [S12] | 相比 GB200 NVL72，单机架 HBM 容量提升约 53%，带宽提升约 174%。 |
| AMD MI350 | 288GB HBM3E，8TB/s 带宽 [S9] | 非 NVIDIA 加速器同样在大内存规格上竞争。 |

这张表支持三个 2027 年判断：

1. **HBM bit 需求增长会快于 GPU 台数增长。** TrendForce 预计 2026 年 AI server 出货量增长超过 20%，而 HBM 消耗量增长超过 70%，驱动因素包括单芯片 HBM 容量上升以及 GPU 和 ASIC 更广泛采用 HBM [S13]。这对 2027 年有参考意义：即使加速器台数增速低于 2025-2026 年，HBM bit 需求仍可能因为每颗旗舰加速器搭载更多 stack、更大容量和更高带宽而更快复合增长。
2. **HBM4/HBM4E 供应成为战略瓶颈，而不是普通 BOM 项。** Micron 已在 2026 年一季度开始为 NVIDIA Vera Rubin 量产出货 36GB 12H HBM4，带宽超过 2.8TB/s，较其 HBM3E 参考的能效提升超过 20% [S14]。Samsung 已开始 HBM4 量产，并称 2026 年 HBM 销售额将超过 2025 年的三倍，HBM4E 将于 2026 年下半年送样，custom HBM 样品将于 2027 年交付客户 [S15]。这些不是普通存储周期数据，而是与加速器平台直接绑定的协同设计路线图。
3. **2027 年组合更偏向内存丰富的推理机架和 ASIC，而不仅是更大的训练超算。** 前沿模型训练和国家级实验室仍然需要训练集群，但商业收入池越来越来自以低延迟和可接受 token 成本服务大量推理请求。这会增加对 HBM 容量、HBM 带宽、机架级 NVLink 或同类互联、CPU 侧内存、快速存储和推理软件的需求。

## 投资与供应链解读

半导体层面的讨论不应被简化为“test-time scaling 会降低 GPU 需求”。更多推理算力会降低单位 token 成本，但更低成本也会扩大使用量。净效应更可能接近 Jevons 效应：更便宜的 reasoning 打开更多 agentic 工作流，反而消耗更多总 token 和更多由 HBM 支撑的加速器小时。

对 2027 年，我会采用以下方向性假设：

- **GPU / 加速器台数：** 仍增长，但增量需求越来越由推理利用率而不只是前沿训练运行来证明。
- **HBM bit 和 HBM 单机价值量：** 增速快于 GPU 台数；产品组合从 HBM3E 转向 HBM4，并开始进入 HBM4E。
- **先进封装：** 更紧，因为 HBM stack 数、base die 复杂度、interposer 面积和热约束都会上升。
- **网络和机架级系统：** 更强，因为 reasoning 工作负载需要共享内存域、prefill / decode 解耦和低延迟 scale-up fabric。
- **Custom ASIC：** 对 hyperscaler 更重要，尤其是当推理流量稳定到足以围绕内部模型族优化时。

主要受益者不只是 GPU 厂商。HBM 供应商、先进封装产能、基板 / interposer 供应商、高端测试、电力 / 冷却厂商和推理软件栈都会获得更高议价权。相反，只绑定通用训练 FLOPS、但缺少内存带宽、服务软件或机架级集成能力的公司，2027 年配置吸引力较弱。

## 什么会证伪这一判断

三个变化会削弱本 thesis：

1. 合成数据和多模态预训练数据打开新的、巨大且干净的高回报数据前沿，使预训练数据扩展重新变得更有吸引力。
2. 算法效率大幅压缩推理轨迹，导致即使 agentic 工作负载增加，输出 token 增长也明显放缓。
3. HBM4 供应扩张快于预期，使瓶颈重新转向 GPU die、电力交付或数据中心容量。

当前证据并不支持这些作为基准情形。更强的信号是：reasoning models、agentic products 和下一代加速器正在围绕更大的推理预算和更高内存带宽进行协同设计。

## 底线

我支持该命题的有条件版本：到 2026 年，对推理密集型前沿 AI 而言，增加 test-time compute 已经成为比单纯增加预训练数据更高回报的边际能力杠杆。到 2027 年，这应推动 AI 硬件需求转向更高的 HBM/GPU 配比：GPU 需求仍强，但 HBM 容量、HBM 带宽和以内存为中心的机架架构会成为更紧的约束。我的基准判断是，2027 年 HBM bit 需求增速约为加速器台数增速的 1.5-2.0x，且推理和 agentic serving 将占新增部署加速器小时的多数。
